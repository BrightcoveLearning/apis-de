Einführung

Die Wiedergabe-API hat vier Operationen:

  • Holen Sie sich ein Video von id
  • Holen Sie sich ein Video nach Referenz-ID
  • Holen Sie sich eine Playlist nach ID
  • Holen Sie sich eine Playlist nach Referenz-ID
Die Wiedergabe-API ist für die clientseitige Verwendung durch Webseiten oder mobile Apps vorgesehen. Verwenden Sie für die allgemeine Medienverwaltung oder die Integration von Video Cloud in Ihre Systeme die CMS-API stattdessen.

Wenn Sie nur Informationen zu dem Video wünschen, das sich gerade im Player befindet, können Sie das mediainfo -Objekt, anstatt einen Playback-API-Aufruf zu tätigen.

Die mediainfo -Objekt enthält Informationen zu dem derzeit im Player geladenen Video, während die Playback-API Informationen zu jedem Video abrufen kann. Einzelheiten finden Sie im Video-Metadaten von mediainfo dokumentieren.

Authentifizierung

Playback-API-Anforderungen werden mit einer verschlüsselten Richtlinienschlüsselzeichenfolge authentifiziert. Der CodePen unten enthält einen Richtlinienschlüssel für ein Brightcove-Beispielkonto, den Sie bei Bedarf einfach verwenden können. Sie können Richtlinienschlüssel für Ihr Konto abrufen, indem Sie entweder die Richtlinien-API oder von einer Ihrer Brightcove-Player.

Anweisungen

Verwenden Sie den folgenden CodePen, um die API zu erkunden:

  1. Stellen Sie zunächst einfach Video- und Playlist-Anfragen für ein Brightcove-Beispielkonto, um die Ergebnisse anzuzeigen.
  2. Versuchen Sie als Nächstes, Ihren eigenen Richtlinienschlüssel zu erhalten, indem Sie entweder die Richtlinien-API oder von einer Ihrer Brightcove-Player.
  3. Ersetzen Sie im CodePen-Formular unten die Konto-ID, den Richtlinienschlüssel, die Video-ID und die Playlist-ID durch Ihre eigenen Werte.
  4. Versuchen Sie, das CSS zu ändern, um die Anzeige der Videodaten anzupassen.
  5. Versuchen Sie, das JavaScript zu ändern, um Videos und Playlists anhand der Referenz-ID statt der ID abzurufen.
