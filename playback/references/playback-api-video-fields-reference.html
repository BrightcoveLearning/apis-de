Primärfelder

Tabelle 1: Primäre Felder
Gebiet Typ Standard Beschreibung
id Zeichenfolge keiner Die Video-ID
account_id Zeichenfolge keiner Die ID des Kontos
created_at ISO 8601 Datum-Uhrzeit-String keiner Das Datum, an dem ein Video dem Konto hinzugefügt wurde; Beispiel: "2014-12-09T06:07:11.877Z"
cue_points Anordnung von cue_point-Objekte leeres Array Markierungen für Mid-Roll-Anzeigenanfragen oder andere Aktionen, die über die Player-API erstellt werden sollen
custom_fields Objekt Leeres Objekt Zuordnung benutzerdefinierter Feldname:Wert-Paare; nur Felder mit Werten sind enthalten
description Zeichenfolge Null Die kurze Beschreibung des Videos - maximal 250 Einzelbyte-Zeichen
duration Nummer Null Länge des Videos in Millisekunden
economics Zeichenfolge AD_SUPPORTED Gibt an, ob Anzeigenanfragen für das Video zulässig sind. Werte: AD_SUPPORTED oder FREE
labels Reihe von Zeichenfolgen Null Anordnung von Etiketten dem Video zugeordnet
link Link-Objekt Null Ein verwandter Link
long_description Zeichenfolge Null Maximal 5000 Einzelbyte-Zeichen
name Zeichenfolge keiner Videotitel - Pflichtfeld (maximal 256 Einzelbyte-Zeichen)
poster Zeichenfolge Null Posterbild-URL
poster_sources Array [] Ein Array von Objekten, die den Postern für das Video entsprechen. Derzeit gibt es nur ein Poster und somit ein Objekt. Das Objekt hat eine einzelne Eigenschaft src deren Wert die URL für das Posterbild ist.
published_at ISO 8601 Datum-Uhrzeit-String keiner Das Datum, an dem ein Video veröffentlicht wurde; Beispiel: "2014-12-09T06:07:11.877Z"
reference_id Zeichenfolge Null Jeder Wert, der innerhalb des Kontos eindeutig ist (maximal 150 Einzelbyte-Zeichen)
sources Array Leeres Array Anordnung von Videoquelle Objekte
tags Array Leeres Array Array von Tags (Strings), die dem Video hinzugefügt wurden (maximal 128 Einzelbyte-Zeichen pro Tag)
text_tracks Anordnung von text_track-Objekte Leeres Array Daten für mit dem Video verknüpfte WebVTT-Untertitel
thumbnail Zeichenfolge Null Die Miniaturbild-URL
thumbnail_sources Array [] Ein Array von Objekten, die den Miniaturansichten für das Video entsprechen. Derzeit gibt es nur ein Thumbnail und somit ein Objekt. Das Objekt hat eine einzelne Eigenschaft src deren Wert die URL für das Miniaturbild ist.
updated_at ISO 8601 Datum-Uhrzeit-String keiner Das Datum, an dem ein Video zuletzt geändert wurde;[1-2] Beispiel: "2015-01-13T17:45:21.977Z"
variants Array [] Eine Reihe von Objekten mit mehrsprachigen Metadaten für die Videos

Anmerkungen

  • [1-2] Die Änderung kann von einem Benutzer oder dem Video Cloud-System vorgenommen worden sein

Tabelle 5: Textspurfelder
Gebiet Typ Standard Beschreibung
src Zeichenfolge (URL) Null URL, unter der sich die .vtt-Datei befindet
srclang ISO-639-1-Sprachcode Null Sprache für die Bildunterschriften

Anmerkungen

  • [9-1] Das Festlegen des mime_type ist optional, aber die Untertitel sollten UTF-8-codiert sein und einen mime_type von "text/vtt" haben.

key_systems Objekt

Das Objekt key_systems enthält eine Liste von Objekten, eines für jeden DRM-Schlüsselnamen. Wenn dieses Objekt definiert ist, ist seine Quelle inhaltsgeschützt.

Tabelle 4: key_systems-Objekt
Gebiet Typ Standard Beschreibung
Schlüsselname Objekt Null Name des Objekts, das den DRM-Typ angibt – gültige Schlüssel sind:

* Fair Play: com.apple.fps.1_0
* Spielbereit: com.microsoft.playready
* Widevine: com.widevine.alpha
FairPlay verwendet key_request_url und certificate_url
andere verwenden license_url		 Zeichenfolge Null URL für den Standort Ihrer DRM-Lizenz oder Ihres DRM-Zertifikats
 

Beispiel 1:

    "key_systems": {
      "com.apple.fps.1_0": {
        "key_request_url": "https://....",
        "certificate_url": "https://..."
      }
    }

Beispiel 2:

    "key_systems": {
      "com.microsoft.playready": {
        "license_url": "https://..."
      },
      "com.widevine.alpha": {
        "license_url": "https://..."
      }
    },

variant Objekt

jede einzelne variant enthält eine Version von Video-Metadaten für eine bestimmte Sprache.

Tabelle 5: Variante Objekt
Gebiet Typ Standard Beschreibung
language Zeichenfolge keiner Die Sprache für die Variante in der Form Sprache-LAND, z. B. ja-JA
name Zeichenfolge keiner Der übersetzte Titel
description Schnur keiner Die übersetzte Beschreibung
long_description Schnur keiner Die übersetzte long_description
custom_fields Objekt {} Eine Karte von Schlüssel-Wert-Paaren für benutzerdefinierte Felder

Beispiel

{
  "language": "ja-JA",
  "name": "バーゼル：ライン川",
  "description": "スイス、バーゼルの中心部にあるライン川のパノラマビュー。",
  "long_description": "スイス、バーゼルの中心部にあるライン川のパノラマビュー。",
  "custom_fields": {
    "language": "Japanese"
  }
}
