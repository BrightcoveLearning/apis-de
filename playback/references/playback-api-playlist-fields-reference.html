Primärfelder

Tabelle 1: Primäre Felder
Gebiet Typ Standard Beschreibung
id Schnur keiner die Playlist-ID
account_id Schnur keiner die ID des Kontos
created_at ISO 8601 Datum-Uhrzeit-Zeichenfolge keiner Datum-Uhrzeit-Video wurde dem Konto hinzugefügt; Beispiel: "2014-12-09T06:07:11.877Z"
description Schnur null die kurze Beschreibung des Videos - maximal 128 Einzelbyte-Zeichen
favorite boolesch false ob die Playlist in den Favoriten enthalten ist
limit Nummer 100 Anzahl der Videos, die die Playlist enthalten kann (integer, 1 - 100) nur für Smart Playlists zurückgegeben
name Schnur keiner Titel der Playlist - Pflichtfeld
reference_id Schnur null jeder Wert, der innerhalb des Kontos eindeutig ist
search Schnur null Suchzeichenfolge, die verwendet wird, um die Videos für eine Smart Playlist zu definieren (Beispiel: +tags:"bird","birds")
type Schnur die Art der Wiedergabeliste Siehe Anmerkung[1] unter
updated_at ISO 8601 Datum-Uhrzeit-Zeichenfolge keiner Datum-Uhrzeit-Wiedergabeliste wurde zuletzt geändert; Beispiel: "2015-01-13T17:45:21.977Z"
videos Array Anordnung von Videoobjekte für die Videos in der Playlist Nur die ersten 20 Videos werden für Playlists zurückgegeben

Anmerkungen

  • [1]

    Die type Wert kann nur einer der folgenden sein: AUSDRÜCKLICH, ACTIVATED_OLDEST_TO_NEWEST , ACTIVATED_NEWEST_TO_OLDEST , ALPHABETICAL , PLAYS_TOTAL , PLAYS_TRAILING_WEEK , START_DATE_OLDEST_TO_NEWEST,START_DATE_NEWEST_TO_OLDEST

    EXPLICIT bedeutet, dass es sich um eine manuelle Wiedergabeliste handelt; alle anderen Typen sind intelligente Wiedergabelisten mit Namen, die ihrem Reihenfolgeschema entsprechen.
