Einführung

Anmerkungen:
  • Der Richtlinienschlüssel wird nur verwendet, um Videos zu steuern, die in Video Cloud gespeichert sind. Wenn Sie den eigenständigen Brightcove-Player verwenden, müssen Sie nicht auf den Richtlinienschlüssel zugreifen oder diesen abrufen.
  • Richtlinienschlüssel, die für neue Brightcove-Player generiert werden, sind standardmäßig nicht für Suche, Domäneneinschränkung, Geofilterung aktiviert oder erfordern SSAI. Sie müssen dies manuell mithilfe der Richtlinien-API (oder dieser Beispiel-App) festlegen. Einzelheiten finden Sie im Übersicht über die Richtlinien-API dokumentieren.
Dieses Beispiel und der zugehörige Code dienen als Leitfaden für Ihre Produktionsentwicklung. Der Code ist weder für den Produktionsgebrauch zertifiziert noch offiziell unterstützt.

Zugangsdaten abrufen

Um die Richtlinien-API zu verwenden, benötigen Sie die entsprechenden Anmeldeinformationen. In dieser Schnellstartanleitung arbeiten Sie mit einem Brightcove-Testkonto, sodass Sie diesen Schritt nicht ausführen müssen. Beachten Sie jedoch, dass Sie Kundenanmeldeinformationen abrufen müssen, wenn Sie mit Ihrem eigenen Konto arbeiten.

    Sie können Anmeldeinformationen auch mithilfe der OAuth-API abrufen, indem Sie die Anweisungen in einem der unten aufgeführten Anleitungen befolgen:

    • OAuth: Client-Anmeldeinformationen mit CURL abrufen
    • OAuth: Erhalten Sie Kundenanmeldeinformationen mit Postman

Richtlinienschlüssel-App erstellen

Hier ist eine einfache App, mit der Sie einen grundlegenden Richtlinienschlüssel erstellen können. Es enthält Berechtigungen zum Durchsuchen der Playback-API.

Siehe den Stift Richtlinienschlüssel-App erstellen von Brightcove Learning Services (@rcroooks1969 ) An CodePen.

Verwenden des CodePens

Hier sind einige Tipps, um den obigen CodePen effektiv zu verwenden:

  • Schalten Sie die aktuelle Anzeige des Players um, indem Sie auf klicken Ergebnis Taste.
  • Drücke den HTML/CSS/JS Tasten, um EINEN der Codetypen anzuzeigen.
  • Klicken Auf CodePen bearbeiten in der oberen rechten Ecke, um diesen CodePen Ihrem eigenen Konto zuzuordnen.
  • Hier finden Sie den gesamten Code, der mit diesem Beispiel verknüpft ist GitHub-Repository.

Proxy-Code

Um Ihre eigene Version der Beispiel-App auf dieser Seite zu erstellen, müssen Sie Ihren eigenen Proxy erstellen und hosten. (Die von Brightcove Learning Services verwendeten Proxys akzeptieren nur Anfragen von Brightcove-Domänen.) Ein Beispielproxy, das dem von uns verwendeten sehr ähnlich ist, jedoch ohne die Überprüfungen, die Anfragen von Nicht-Brightcove-Domains blockieren, finden Sie in dieses GitHub-Repository. Dort finden Sie auch grundlegende Anweisungen zur Verwendung und eine ausführlichere Anleitung zum Erstellen von Apps rund um den Proxy in Verwenden der REST-APIs.
