Die Brightcove Syndication Feed-API ist eine öffentliche API zum Abrufen von MRSS-Feeds, die mit dem Syndication-Konfigurations-API.

Die Syndication Configuration API kann auch zum Abrufen von Feeds verwendet werden, indem einfach das syndication_url im Syndication-Objekt an einen Browser zurückgegeben, aber die Feed-API unterscheidet sich dadurch, dass sie keine Authentifizierung erfordert - jeder mit der richtigen URL kann den Feed abrufen.

Die Hauptanwendungsfälle für das Abrufen von Feeds mit der einen oder anderen API sind wie folgt:

  • Wenn Sie den Feed nur selbst hosten möchten, beispielsweise für Abonnenten, verwenden Sie die Syndication Configuration API, um den Feed abzurufen.
  • Wenn Sie Syndication-Partner haben, denen Sie den Feed auf beliebige Weise verwenden und hosten lassen möchten, können Sie die Feed-API-URL mit dem syndication_id um Feeds direkt abzurufen.

API-Referenz / Basis-URL

Die Feed-API-Referenz enthält alle Details zu den verfügbaren Anfragen und Antworten.

Die Basis-URL lautet:

https://social.feeds.brightcove.com/v1

Authentifizierung

Der Zugriff auf die Feed-API funktioniert nicht erfordern eine Authentifizierung, da Feeds öffentlich zugänglich sein sollen.

Betrieb

Es gibt nur einen Vorgang für die Feed-API: Holen Sie sich einen Feed basierend auf dem syndication_id.

Anfrage-URL

    https://social.feeds.brightcove.com/v1/accounts/{account_id}/mrss/syndications/{syndication_id}/feed

Methode: WERDEN

Abfrageparameter

Alle Abfrageparameter sind optional.

limit
Maximale Anzahl von Elementen, die in die Feedantwort aufgenommen werden sollen. Zulässiger Bereich von 1 bis 100. Standard: 100.
offset
Anzahl der Elemente, die beim Generieren des zurückgegebenen Feeds übersprungen werden sollen. Standard: 0.
updated_after
Eine ISO-8601-Datetime-Zeichenfolge (z. B. 2016-03-16T15:42:06.000Z oder 2016-03-16T10:47:29-05:00 ), die die Feed-Ergebnisse auf Assets beschränkt, die nach dem angegebenen Datum aktualisiert wurden.

Anfragetext: keiner

Das ist wichtig! Wenn Sie einen Anfragetext einschließen, wird ein Fehler zurückgegeben.

Antworttyp: XML
