Einführung

Mit der In-Page-Erlebnis-API können Sie In-Page-Erlebnisse programmgesteuert verwalten, was es besonders für CMS-Integrationen nützlich macht, damit Benutzer Erlebnisse von ihrem eigenen CMS aus erstellen können, anstatt zur Galerie in Studio zu wechseln.

Dieser Schnellstart enthält eine Beispielimplementierung, die Benutzern Folgendes ermöglicht:

  1. Holen Sie sich alle In-Page Experience-Vorlagen für ein Konto
  2. Erstellen Sie ein Basiserlebnis aus der ausgewählten Vorlage
  3. Wählen Sie entweder eine Playlist oder eine Liste von Videos aus, die Sie dem Erlebnis hinzufügen möchten
  4. Aktualisieren Sie das Erlebnis mit den ausgewählten Videos/Playlists
  5. Veröffentlichen Sie die Erfahrung

Unten sehen Sie eine allgemeine Ansicht der App-Logik.

Verwenden des CodePens

Hier sind einige Tipps, um den obigen CodePen effektiv zu verwenden:

  • Schalten Sie die aktuelle Anzeige des Players um, indem Sie auf klicken Ergebnis Taste.
  • Drücke den HTML/CSS/JS Tasten, um EINEN der Codetypen anzuzeigen.
  • Klicken Auf CodePen bearbeiten in der oberen rechten Ecke, um diesen CodePen Ihrem eigenen Konto zuzuordnen.
  • Hier finden Sie den gesamten Code, der mit diesem Beispiel verknüpft ist GitHub-Repository.

Proxy-Code

Um Ihre eigene Version der Beispiel-App auf dieser Seite zu erstellen, müssen Sie Ihren eigenen Proxy erstellen und hosten. (Die von Brightcove Learning Services verwendeten Proxys akzeptieren nur Anfragen von Brightcove-Domänen.) Sie können zwei Versionen unseres Proxy-Codes herunterladen:

  • Dies ist eine allgemeine Version, die erwartet, dass Client-Anmeldeinformationen mit der Anfrage übergeben werden
  • Mit dieser Version können Sie Ihre Client-Anmeldeinformationen im Proxy selbst in den Zeilen 25-26 speichern (empfohlen)
