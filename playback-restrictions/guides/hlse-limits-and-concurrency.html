Diese Funktion ist für eine bestimmte Gruppe von Kunden mit Zugriff auf die eingeschränkte Verfügbarkeitsphase der Brightcove-Wiedergabebeschränkungen verfügbar.

Einführung

Wenn Sie Ihre Inhalte pro Benutzer begrenzen möchten, aber kein DRM integrieren müssen oder möchten, dann kann die Verwendung von HLSe die Lösung für Sie sein.

Mit der Standardverschlüsselung mit AES-128 (HLSe) können Sie Ihre Inhalte über das Internet verschlüsseln. Es ist nicht so sicher wie die Verwendung von DRM, da es einen öffentlich zugänglichen Entschlüsselungsschlüssel verwendet.

Verwendung von HLSe mit Lizenzschlüsselschutz von Brightcove , ermöglicht Ihnen, Wiedergabeanforderungen basierend auf Benutzer- oder erweiterten Informationen zu autorisieren, um den Zugriff auf die Verschlüsselungsschlüssel zuzulassen oder zu verweigern und die Sicherheit der Standardverschlüsselung zu verbessern.

License Key Protection konzentriert sich auf den Schutz für DRM-Lizenzanfragen sowie die AES-128-Entschlüsselung. JWT-Ansprüche ermöglichen es dem Lizenzschlüsselschutz, Einschränkungen auf Benutzerebene zu unterstützen. Das bedeutet, dass Sie die Wiedergabe auf einen User-Agent, eine maximale Anzahl verschiedener IP-Adressen oder eine maximale Anzahl von Benutzern für jeden Token beschränken können.

Wenn Sie diese Funktion noch nicht kennen, lesen Sie die Überblick: Brightcove-Wiedergabebeschränkungen dokumentieren.

Einstieg

Richten Sie Ihr Konto ein und bereiten Sie Ihre Inhalte vor.

  1. Wenden Sie sich an Ihren Account Manager, um Ihr Konto für HLSe und Wiedergabebeschränkungen zu aktivieren.
  2. Sobald HLSe für Ihr Konto aktiviert ist, nehmen Sie die Videos auf, die Sie schützen möchten, oder transcodieren Sie sie erneut.

Arbeitsablauf

Führen Sie die folgenden Schritte aus, um Gerätelimits festzulegen oder gleichzeitig mit HLSe zu streamen:

  1. Ein ... kreieren JSON-Webtoken (JWT) pro Benutzer.

    Für jede Benutzersitzung müssen Sie ein JWT erstellen. Diese soll anhand der gewünschten Restriktionen mit unterschiedlichen Ansprüchen generiert werden.

    • Gerätebeschränkungen finden Sie im Gerätelimits einstellen Sektion.

    • Informationen zur Stream-Parallelität finden Sie im Stream-Parallelität einstellen Sektion.

  2. Konfigurieren Sie Ihren Player so, dass er Lizenzschlüsselschutz verwendet.

    Einzelheiten finden Sie im Verwenden des Lizenzschlüsselschutzes Abschnitt der Übersicht: Dokument zu Brightcove-Wiedergabeeinschränkungen.

Gerätelimits einstellen

Mit License Key Protection können Sie die Wiedergabe auf eine bestimmte Anzahl von Geräten pro Benutzer beschränken.

Behauptungen

  • :maxip

    Für HLSe-Inhalte ist die maxip Claim definiert, wie viele verschiedene IP-Adressen gleichzeitig streamen können.

Beispiel

In diesem Beispiel wird ein JWT erstellt, um die Wiedergabe auf eine IP-Adresse pro Benutzer (ein Gerät pro Benutzer) zu beschränken:

  1. Generieren Sie ein öffentlich-privates Schlüsselpaar.

    Einzelheiten finden Sie im Generieren Sie ein öffentlich-privates Schlüsselpaar Abschnitt des Dokuments Erstellen eines JSON Web Token (JWT)-Dokuments.

  2. Registrieren Sie Ihren öffentlichen Schlüssel.

    Einzelheiten finden Sie im Verwenden von Authentifizierungs-APIs dokumentieren.

  3. Erstellen Sie ein JSON-Webtoken.

    Hier ist ein Beispiel mit bash. Stellen Sie sicher, dass Sie die hinzufügen "maxip":1 behaupten, die Wiedergabe auf eine IP-Adresse (Gerät) pro Benutzer zu beschränken.

    #! /usr/bin/env bash
  # Static header fields.
  HEADER='{
    "type": "JWT",
    "alg": "RS256"
  }'
  
  payload='{
    "pkid": "your policy key",
    "accid": "your account id",
    "maxip": 1
  }'
  
  # Use jq to set the dynamic `iat` and `exp`
  # fields on the payload using the current time.
  # `iat` is set to now, and `exp` is now + 1 second.
  PAYLOAD=$(
    echo "${payload}" | jq --arg time_str "$(date +%s)" \
    '
    ($time_str | tonumber) as $time_num
    | .iat=$time_num
    | .exp=($time_num + 60 * 60)
    '
  )
  
  function b64enc() { openssl enc -base64 -A | tr '+/' '-_' | tr -d '='; }
  
  function rs_sign() { openssl dgst -binary -sha256 -sign {your_private_key_path} ; }
  
  JWT_HDR_B64="$(echo -n "$HEADER" | b64enc)"
  JWT_PAY_B64="$(echo -n "$PAYLOAD" | b64enc)"
  UNSIGNED_JWT="$JWT_HDR_B64.$JWT_PAY_B64"
  SIGNATURE=$(echo -n "$UNSIGNED_JWT" | rs_sign | b64enc)
  
  echo "$UNSIGNED_JWT.$SIGNATURE"
