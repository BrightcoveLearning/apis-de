Inhalt der Seite

    Implementierung von Bereitstellungsregeln

    In diesem Thema erfahren Sie, wie Sie die Lieferregeln von Brightcove implementieren, mit denen Sie anpassen können, wie Ihre Medien bereitgestellt werden, um Ihre Geschäftsziele zu erreichen.
    Diese Funktion steht für eine bestimmte Gruppe von Kunden mit Zugriff auf die Phase der begrenzten Verfügbarkeit von Bereitstellungsregeln zur Verfügung.

    Übersicht

    Verlage müssen häufig die inhaltlichen Merkmale ihrer Medienmanifeste sowohl aus technischen als auch aus geschäftlichen Gründen ändern. Um diesen Anforderungen gerecht zu werden, hat Brightcove einen Rahmen geschaffen, in dem Regeln angewendet werden können, die den Inhalt des von Dynamic Delivery zurückgegebenen Manifests ändern.

    Lieferregeln bestehen aus Bedingungen und Aktionen.

    • Bedingungen - „If“ -Bedingungen, die eine bestimmte Aktion auslösen
    • Aktionen - „Dann“ -Parameter, die definieren, wie das Manifest geändert wird

    Weitere Informationen zu Lieferregeln finden Sie unter den folgenden Themen:

    • Übersicht: Regeln für die Lieferung
    • Verwenden von Lieferregeln mit Brightcove Player
    • Verwenden von Bereitstellungsregeln mit den Native SDKs
    • Überblick über die dynamische Lieferung

    Konto-Setup

    Wenden Sie sich an Ihren Account Manager, um mehr über die Preise zu erfahren und diese Funktion für Ihr Konto zu aktivieren.

    Regeln anwenden

    Das folgende Flussdiagramm zeigt, wie Lieferregeln angewendet werden.

    Bedingungen

    Die in der ersten Phase unterstützten „if“ -Bedingungen umfassen Folgendes:

    • Explizite Anfrage - Eine Konfigurations-ID ist die ID einer Aktion, die Sie erstellt haben. Sie verwenden diese ID, um zur Laufzeit an die Playback-API zu übergeben. Diese ID wird Konfigurationswerten zugeordnet, die es zugrunde liegenden Diensten ermöglichen, Entscheidungen zu treffen, die von der Playback-API genutzt werden, um die entsprechenden Antwortdaten zurückzugeben.

    • Gerätegruppe - Gerätetyp, wie vom Analysieren des Benutzeragenten erkannt. Zu den derzeit unterstützten Gruppen gehören:
      • Android
      • appletv
      • iPad
      • iPhone
      • desktop-chrom
      • Desktop-Firefox
      • Desktop-Safari
      • fetchtv
      • ios-andere
      • roku-7
      • smartphone-generisch
      • Tablet-android
      • Tablet-Kindle
      • unbekannt
       
    • Geographie - Physischer Standort des anfragenden Geräts:
      • Kontinent
      • Land

    Aktionen

    Die in der ersten Phase unterstützten Aktionen oder „dann“ -Parameter umfassen Folgendes:

    • Media Delivery CDN - Von den CDNs, die für die Verwendung innerhalb eines bestimmten Dynamic Delivery-Kontos konfiguriert sind, wird das CDN für die Bereitstellung von Mediensegmenten verwendet.

    • Wiedergabe-Merkmale - Filter basierend auf Merkmalen der zugrunde liegenden Medienwiedergaben. Hierzu zählt Folgendes:
      • Minimale Videobitrate
      • Maximale Videobitrate
      • Minimale Videoauflösung
      • Maximale Videoauflösung
      • Maximale Anzahl von Audio-Wiedergaben
      • Maximale Anzahl von Videowiedergaben
      • Erste Videobitrate
      • Diskontinuitäten

    Scope

    Aktionen können auf zwei Ebenen angewendet werden:

    • Konto - Aktionen, die für alle Anfragen gelten, die an das Konto eines bestimmten Kunden gestellt werden
    • Spezifische Anfrage - Aktionen, die für eine bestimmte Anfrage aufgerufen werden

    Aktionen werden in der oben aufgeführten Reihenfolge angewendet. Spätere Aktionen werden frühere überschreiben.

    Definieren von Regeln

    Verwenden Sie die API für Bereitstellungsregeln, um die Medienbereitstellung anzupassen.

    API für Lieferregeln

    Mit der API „Delivery Rules “ können Sie Bedingungen und Aktionen definieren, um Ihre Medienbereitstellung zu steuern.

    Basis-URL

    Die Basis-URL für die API lautet: 

    https://delivery-rules.api.brightcove.com

    Konto-Pfad

    In allen Fällen werden Anfragen für ein bestimmtes Video Cloud-Konto gestellt. Daher fügen Sie immer den Begriff Konten gefolgt von Ihrer Konto-ID zur Basis-URL hinzu: 

    https://delivery-rules.api.brightcove.com/accounts/{accountID}

    Autorisierung

    Ein Zugriffstoken für Anfragen ist erforderlich und muss im Authorization-Header vorhanden sein: 

    Authorization: Bearer {access_token}

    Das Zugriffstoken ist ein temporäres OAuth2-Zugriffstoken, das vom Brightcove OAuth Service bezogen werden muss. Weitere Informationen zum Abrufen von Client-Anmeldeinformationen und zum Abrufen von Zugriffstoken finden Sie in der Brightcove OAuth Overview.

    Berechtigungen

    Anfragen an die Playback Rights API müssen von Client-Anmeldeinformationen mit den folgenden Berechtigungen gestellt werden:

    • video-cloud/delivery-rules/read
    • video-cloud/delivery-rules/all

    Regeln verwalten

    Die API für Lieferregeln unterstützt die folgenden Anfragen. Einzelheiten zur API finden Sie in der API-Referenz für Bereitstellungsregeln.

    GET /accounts/{accountID}

    Verwenden Sie die folgende PUT-Anfrage, um die Bedingungen für ein Konto zu aktualisieren. 

    PUT /accounts/{accountID}/conditions
  Content-Type: application/json
  Authorization: Bearer {access_token}
  Body: {conditions object}

    Anfragetext für Bedingungen

    Hier ist der Anfragetext für Bedingungen.

    Feld Art Beschreibung
    name String Eindeutige Kennung für die Bedingung
    device_group String Array von Gerätetypen, wie vom Analysieren des Benutzeragenten erkannt
    request_country String Array von zweibuchstaben Ländercodes
    request_continent String Array von zweibuchstaben Kontinenten Codes
    then String Eindeutige Kennung für die zugehörige Aktion

    Einschränkungen

    Bei der Verwendung von Lieferregeln gibt es einige bekannte Einschränkungen:

    • Medieninhalte - Lieferregeln funktionieren nicht mit Smooth- oder PMP4-Videos.
    • Nur Audio - Regeln für die Bereitstellung mit aktivierten serverseitigen Anzeigen (SSAI) erfordern sowohl Video- als auch Audioinhalte. Dies ist eine SSAI-Beschränkung.
