Einführung

Der Playback Rights Management Service von Brightcove bietet eine skalierbare und ausdrucksstarke Möglichkeit zur Verwaltung der Videowiedergabe. Dieser Service umfasst die folgenden Funktionen:

  • Beschränken Sie die Wiedergabe nach Region
    • Land
    • Postleitzahl
    • Designiertes Marktgebiet (DMA)
  • Proxy-Einschränkungen definieren
  • Definiere IP-Einschränkungen für jedes Video
  • Domaineinschränkungen definieren
  • Legen Sie wiederkehrende Zeitpläne fest
  • Anzeige nach Gerät einschränken
  • Begrenzen Sie die Anzahl der Videos, die ein Zuschauer gleichzeitig ansehen kann

Spezifikationen

Der Playback Rights Management Service von Brightcove bietet die folgenden Funktionen:

Wiedergaberechte

Mit den Wiedergaberechten können Sie Einschränkungen für die Videowiedergabe festlegen. Diese Rechte werden in der API für Wiedergaberechte und jedem Set wird eine eindeutige ID zugewiesen (playback_right_id). Ein Satz von Rechten kann null oder mehr Videos zugeordnet werden.

Wenn ein Video ein hat playback_right_id , werden die im Playback Rights-Objekt angegebenen Einschränkungen auf ALLE Anfragen für dieses Video angewendet.

Die Wiedergaberechte umfassen die folgenden Kategorien:

Geographie

Mit geografischen Einschränkungen können Sie steuern, wo Ihre Inhalte angezeigt werden. Sie können Einschränkungen basierend auf den folgenden Punkten festlegen:

  • Land
  • Postleitzahl (nur USA)
  • Designated Market Area (DMA - ein von Nielson entwickeltes Konzept)

Der Geografie-Check wird sowohl beim Start der Videowiedergabe als auch während der Wiedergabe ausgeführt. Wenn die Midstream-Geoprüfung fehlschlägt, stoppt der Player die Videowiedergabe.

Proxy-Einschränkungen

Proxy-Einschränkungen ermöglichen es Ihnen, die Wiedergabe für bestimmte Verbindungstypen zu blockieren.

IP-Adressen und Domänen

Mit Domaineinschränkungen können Sie die Anzeige von Videos auf bestimmte Domains beschränken. IP-Einschränkungen beschränken die Videowiedergabe auf bestimmte IP-Adressen (Konto- oder Videoebene).

Zeitpläne

Mit wiederkehrenden Zeitplänen können Sie die Videowiedergabe für bestimmte Zeiträume einschränken. Sie können beispielsweise zulassen, dass Ihre Inhalte nur am Wochenende abgespielt werden. Ein anderes Beispiel wäre, die Wiedergabe montags, dienstags und freitags zwischen 8:00 und 20:00 Uhr zuzulassen.

Einschränkungen auf Benutzerebene

Sie haben die Möglichkeit, für jeden Betrachter eine eindeutige ID pro Konto zu erstellen. Für Einschränkungen auf Benutzerebene müssen Sie bei jeder Anforderung ein JWT-Token verwenden. Damit können Sie folgende Einschränkungen festlegen:

Gerätegrenzen

Die Geräteregistrierung erfolgt, wenn eine DRM-Lizenzanfrage gestellt wird. Jedem Gerät wird eine eindeutige ID zugewiesen. Bei jeder Lizenzanfrage wird das Gerätelimit überprüft und durchgesetzt. Um diese Einschränkung zu verwenden, müssen Sie ein JWT mit der aktuellen Benutzer-ID (uid) generieren.

Beschränkungen für gleichzeitige Streams

Die Begrenzung gleichzeitiger Streams pro Benutzer hält Zuschauer davon ab, ihre Anmeldeinformationen mit Freunden zu teilen, die keine Konten haben. Mit Limits für gleichzeitige Streams definieren Sie die Anzahl der Videostreams, die ein bestimmter Benutzer zu einem bestimmten Zeitpunkt ansehen kann.

Stichworte

Beschränken Sie die Inhalte, die Zuschauer basierend auf der Abonnementberechtigung ansehen können. Das von Ihnen generierte JWT-Token gibt die Tags an, die zur Anzeige zulässig sind. Der Playback Rights Service stellt sicher, dass der Benutzer nur Videos mit den angegebenen Tags ansehen kann.

Angenommen, Sie haben die folgenden Abonnementstufen:

  • Silber – Sieh dir alle Videos an, außer Erstausstrahlungen und Livestreams
  • Gold – alle Videos außer Livestreams ansehen
  • Platin - Sieh dir jedes Video oder jeden Live-Stream an

Verknüpfen Sie zunächst Tags mit Ihren Videos, um zwischen Standard, Erstlauf und Live zu unterscheiden. Geben Sie dann die zutreffenden Tags im JWT-Token an, das Sie für den Benutzer generieren.

Videoeinschränkungen und Wiedergaberechte

Die folgenden Einschränkungen können für Videos über das Medienmodul oder der CMS-API:

  • Geofilterung (Anzeigen in bestimmten Ländern zulassen/verbieten)
  • Planung (Festlegen des Start- und/oder Enddatums für die Videoverfügbarkeit)

Diese Einstellungen sind vom Playback Rights Management-System getrennt, aber ein Video muss Einschränkungen auf Videoebene bestehen, bevor Playback Rights Management-Regeln angewendet werden.
