Einführung

Beim Aufnehmen von Videos können Sie XML-Manifestdateien verwenden, um zusätzliche Metadaten (einschließlich Videotitel, Beschreibung, Laufzeiten, Labels und benutzerdefinierte Metadaten) sowie Miniatur- und Untertiteldateien anzugeben.

Notiz:
  • Das ist ein Beta Aufnahmeservice - wenden Sie sich an Ihren Account Manager, wenn Sie an der Beta-Teilnahme interessiert sind.
  • Obwohl die XML-Tags das Schlüsselwort verwenden ooyala , können Sie mit dieser Aufnahmemethode Videos in Ihr Video Cloud-Konto hochladen.

Umgang mit Metadaten

Wenn Sie einschließen metadata für Videos, wenn Sie sie aufnehmen, wird es auf zwei Arten den Video Cloud-Videofeldern zugeordnet:

  1. Standardmäßig ist jeder metadata Schlüssel/Wert-Paar wird als a . hinzugefügt Schild in der Form: meta::key::value. Zum Beispiel, <ooyala:metadata name="category">cooking</ooyala:metadata> würde das Etikett produzieren meta::category::cooking

Sie können die folgenden Elemente in der XML-Manifestdatei verwenden.

<ooyala:labels> Ein oder mehrere Werte, durch Kommas getrennt. Beispiel: /sports,/sports/jogging. Diese Werte werden zum Ausfüllen verwendet Video Cloud-Tags. Optional

Beispiel: <ooyala:metadata name="category">cooking</ooyala:metadata>.

 Optional
<ooyala:remoteasset> Gibt ein aufzunehmendes Remote-Asset an. Ein Remote-Asset ist ein Video, das an einem anderen Ort als Video Cloud gehostet wird. Geben Sie die folgenden Attribute an:
  • durationInMs: Mediendauer (in Millisekunden)
  • :url URL, unter der sich das Remote-Asset befindet. Die URL kann enthalten http:// oder https://.

Diese Einstellung wird dem Wert der hosted_at-Eigenschaft für ein Remote-Asset zugeordnet. Beispiel: <ooyala:remoteasset durationInMs='5000' url="http://your_website/your_remote_assets.mp4" />

Notiz: Nachdem ein Remote-Asset erstellt wurde, kann sich seine Weitergabe an die verschiedenen CDNs um 60 Sekunden oder mehr verzögern. Wenn Sie ein Remote-Asset zu früh nach seiner Erstellung anfordern, werden die Ergebnisse von den CDNs zwischengespeichert, was einige Minuten dauern kann.
 erforderlich (nur für Remote-Assets)
<ooyala:profileguid> Gibt ein Video Cloud-Aufnahmeprofil an, das für die Verarbeitung des Videos verwendet wird. Wenn Sie das Kontostandardprofil nicht verwenden möchten, müssen Sie das Profil angeben, das Sie stattdessen verwenden möchten. Sehen Verwalten von Aufnahmeprofilen. Optional

Schritt für Schritt

Erstellen einer XML-Manifestdatei

  1. Laden Sie Ihre Videos und anderen Inhalte (Thumbnails, Untertiteldateien usw.) auf Ihren Hosting-Server hoch. Die URLs müssen öffentlich verfügbar sein.
  2. Öffnen Sie einen Texteditor, vorzugsweise einen, der XML validiert. (Das Brightcove Learning Services-Team verwendet Visual Studio-Code aber es gibt noch einige andere gute Optionen).
  3. Fügen Sie die erforderlichen XML-Header hinzu. Zum Beispiel: 
    <?xml version="1.0" encoding="utf-8"?> <rss version="2.0" xmlns:media="http://search.yahoo.com/mrss/" xmlns:dcterms="http:/ /purl.org/dc/terms/" xmlns:fh="http://purl.org/syndication/history/1.0" xmlns:ooyala="http://www.brightcove.com/mrss/">
  4. Ein ... kreieren <channel> Container.
  5. Erstelle ein <item> Eintrag für jedes Video, der die gewünschten Elemente aus der obigen Tabelle enthält.

    Im folgenden Beispiel gibt die XML Metadaten für zwei Videos an. Beim ersten Video befinden sich die Datei und die Miniaturansichten auf einem Upload-Server und werden von diesem Server abgerufen. Für das zweite Video wurden die Datei und das Thumbnail bereits auf einen Ooyala-Server hochgeladen. Das zweite Video hat auch Flugzeiten.

    <?xml version="1.0" encoding="utf-8"?> <rss version="2.0" xmlns:media="http://search.yahoo.com/mrss/" xmlns:dcterms="http:/ /purl.org/dc/terms/" xmlns:fh="http://purl.org/syndication/history/1.0" xmlns:ooyala="http://brightcove.com/mrss"> <channel> <item > <media:content url="http://mysite.com/upload/lacrosse_70.mov" /> <media:thumbnail url="http://mysite.com/upload/lacrosse_70_previewimage.jpg" /> <media: title>Sport, der wirklich existiert: Lacrosse</media:title> <media:description>Meine Beschreibung meines Videos</media:description> <ooyala:labels>/sports/lacrosse,/hobbies/lacrosse</ooyala:labels> <link>http:// mysite.com/videos/sports/real_sports.html</link> <ooyala:metadata name="video_ID">70</ooyala:metadata> <ooyala:metadata name="season_number">1</ooyala:metadata> < /item> <item> <media:content url="curling_71.mov" /> <media:thumbnail url="curling_71_previewimage.jpg" /> <media:title>Sport, der wirklich existiert: Curling</media:title> <media:description>Meine Beschreibung meines Videos</media:description> <ooyala:labels>/sports/curling,/hobbies/curling</ooyala:labels> <link>http:// mysite.com/videos/sports/real_sports.html</link> <ooyala:metadata name="video_ID">71</ooyala:metadata> <ooyala:metadata name="season_number">1</ooyala:metadata> < !-- Flugzeiten einstellen --> <dcterms:valid>start=2011-06-28T15:15:00-07:00; end=2022-12-01T16:00:00-07:00;scheme=W3C-DTF </dcterms:valid> </item> </channel> </rss>
  6. Speichern Sie die Datei als XML. Achten Sie darauf, einen eindeutigen Namen anzugeben. Wenn Sie zwei Dateien mit demselben Namen angeben und eine hochladen, während die andere verarbeitet wird, wird die zweite möglicherweise ignoriert
  7. Laden Sie mit der Upload-Methode Ihrer Wahl die XML-Manifestdatei auf Ihren Hosting-Server hoch, auf dem Sie die Videos hochgeladen haben.
    Denken Sie daran, dass die URL für das XML-Manifest genau mit der übereinstimmen muss, die Sie Ihrem Account Manager bereitgestellt haben.
