Einführung

Sie können Medien-RSS (MRSS) verwenden, damit Brightcove Ihre Inhalte von Ihrem eigenen System aus einem von Ihnen veröffentlichten MRSS-Feed abruft. Video Cloud ruft die Metadaten aus dem MRSS-Feed ab und erstellt entweder ein Remote-Asset oder lädt es herunter (von wo sie gehostet werden) und verarbeitet die Videos und alle zugehörigen Assets (Miniaturansichten, Untertiteldateien usw.).

Wenn Sie daran interessiert sind, an der Beta-Version für die Aufnahme aus einem remote gehosteten MRSS-Feed teilzunehmen, wenden Sie sich an Ihren Account Manager.

Übersicht über MRSS Ingest

Über MRSS

Media RSS (MRSS) ist eine Erweiterung von RSS, mit der Sie Multimediadateien in RSS-Feeds syndizieren können. Technische Details finden Sie im Medien-RSS-Spezifikation.

MRSS einrichten

Wenden Sie sich an Ihren Account Manager und geben Sie an, dass Sie MRSS verwenden möchten. Ihr MRSS-Feed muss auf einer öffentlichen URL gehostet werden und die darin enthaltenen Videos und Assets müssen ebenfalls auf öffentlichen URLs sein. URLs können http:// oder https:// enthalten.

Verwenden von MRSS

Um MRSS zu verwenden, hosten Sie Metadaten und/oder Videos und andere Dateien (Thumbnails, Untertitel) auf einem Webserver und erstellen eine XML-Manifestdatei das auf diesen Inhalt verweist.

Beachten Sie, dass der MRSS-Feed alle fünf Minuten von Brightcove abgerufen wird.

Verwenden von MRSS nur zum Abrufen von Metadaten

  1. Erstellen Sie eine XML-Manifestdatei wie in beschrieben XML-Manifestdatei. Sie müssen die verwenden ooyala:remoteasset -Tag für die reine Metadatenaufnahme.
  2. Hosten Sie die XML-Manifestdatei an einem der von Ihnen angegebenen MRSS-Speicherorte. Stellen Sie sicher, dass Dateiname und Pfad genau einem der Pfade entsprechen, die Sie Brightcove zur Verfügung gestellt haben.

Verwenden von MRSS zum Abrufen von Videos, Miniaturansichten und Untertiteldateien

  1. Hosten Sie eine oder mehrere Dateien (Videos, Miniaturansichten und Untertiteldateien) auf einem Webserver. Die URLs müssen öffentlich zugänglich sein.
  2. Erstelle ein XML-Manifestdatei. Stellen Sie sicher, dass die Einträge in der Datei auf die Dateien auf Ihrem Server verweisen. Insbesondere der Wert der <media:content> Elements href Attribut muss eine HTTP-URL zu der Datei auf Ihrem Server sein, wie im folgenden Beispiel: 
    <media:content url="http://mysite.com/upload/lacrosse_70.mov" />
  3. Hosten Sie die XML-Manifestdatei an einem der von Ihnen angegebenen MRSS-Speicherorte. Stellen Sie sicher, dass Dateiname und Pfad genau einem der Pfade entsprechen, die Sie Brightcove zur Verfügung gestellt haben.
    Notiz: Leider gilt das Konzept der "Client-Protokollierung" nicht für die MRSS-Feed-basierte Verarbeitung. Der beste Weg, um zu überprüfen, ob Ihr Feed erfolgreich verarbeitet wird, besteht darin, das Aussehen Ihrer Assets zu überprüfen, entweder mit Video Cloud Studio oder die Video Cloud CMS-API.
