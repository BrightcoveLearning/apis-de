Einschränkungen

Anmerkungen:
  • Es gelten die in diesem Thema beschriebenen bedingten Ausgabeeinstellungen nur zu älteren Aufnahmeprofilen, nicht zu Dynamic Delivery- oder Context Aware Encoding-Profilen.
  • Wenden Sie sich an den Brightcove-Support, um bedingte Ausgaben mit Dynamic Delivery-Aufnahme/Transcodierung zu verwenden.

Warum Wiedergaben überspringen?

Das Überspringen von Wiedergaben ist nützlich, wenn die Qualität der aufgenommenen Quellvideos stark schwankt. Dies ist beispielsweise häufig bei Nachrichtenorganisationen der Fall, die Videos verwenden, die von Telefonen oder anderen Geräten im Außendienst aufgenommen wurden.

Ohne bedingte Ausgaben erstellt das Dynamic Ingest-System genau so viele Wiedergaben, wie im Profil angegeben sind.

Angenommen, Sie nehmen ein Video mit geringer Qualität (Videobitrate von etwa 820 kbps) mit einem Profil auf, das die folgenden unbedingten Videowiedergaben enthält.

Videowiedergaben
Format Audio Bitrate Video-Bitrate
mp4 64 450
mp4 96 900
mp4 128 1700
HLS 64 450
HLS 96 700
HLS 96 900
HLS 96 1200
HLS 128 1700
HLS 128 2000

Beachten Sie, dass die letzten drei (hervorgehobenen) HLS-Wiedergaben in der Bitrate identisch sind und dass die Bitrate nur geringfügig höher ist als die der nächstbesten HLS-Wiedergabe. Beachten Sie auch, dass der Unterschied in den Bitraten für die zwei besten MP4-Wiedergaben gering ist. Der Grund dafür ist, dass die Transcodierung die Bitrate der Quelle niemals verbessern kann und die Quelle hier eine durchschnittliche Bitrate von etwa 800 kbps hatte, so dass Wiedergaben mit höheren spezifizierten Bitraten etwas in diesem Bereich haben (Sie werden kleine Unterschiede sehen) aufgrund unterschiedlicher Methoden zur Berechnung der durchschnittlichen Bitrate für segmentierte und nicht segmentierte Videoformate).

Die letzten vier Wiedergaben im Satz sind also im Wesentlichen redundant und bieten kaum oder gar keine Qualitätsverbesserung gegenüber den Wiedergaben mit "niedrigerer Qualität", aber sie werden trotzdem erstellt und gespeichert. Dies kann verhindert werden, indem bedingte Ausgaben verwendet werden, die die Generierung einer Wiedergabe verhindern, wenn die Quelle nicht den von Ihnen angegebenen Qualitätskriterien entspricht.

Beachten Sie, dass wir bedingte Ausgabeeinstellungen zum hinzugefügt haben Standardprofile und empfehlen Ihnen, diese Profile nach Möglichkeit zu verwenden.

Implementierung

Sie können hinzufügen skip Einwände gegen einige der Wiedergaben. Die skip -Objekt enthält eine oder mehrere Bedingungen, und wenn diese Bedingungen nicht erfüllt sind, wird die Wiedergabe nicht erstellt.

Standardprofile überspringen automatisch Wiedergaben, wenn die Bitrate des Quellvideos niedriger als die für die Wiedergabe angegebene Bitrate ist. Wenn Sie dieses Verhalten für benutzerdefinierte Profile wünschen, müssen Sie es wie in diesem Thema beschrieben hinzufügen.

Hinweise zur Verwendung

  1. Ein Aufnahmeprofil muss Mindestens eine nicht überspringbare Darstellung einschließen - Wenn Sie versuchen, ein Profil zu erstellen, in dem alle Darstellungen überspringbar sind, erhalten Sie eine Fehlermeldung.
  2. Sie müssen sicherstellen, dass für jeden Inhaltstyp mindestens eine Wiedergabeversion erstellt wird. Zum Beispiel:
    • Wenn Sie sowohl reine Audio- als auch Videowiedergaben erstellen, müssen Sie für jede eine nicht überspringbare Wiedergabeversion haben
    • Wenn Sie DRM verwenden, müssen Sie für jedes Format, das mit diesem DRM-Typ funktioniert (Smooth, HLS, DASH usw.)

Syntax

Die Syntax für die skip Das Objekt, das Sie zu einer Wiedergabespezifikation hinzufügen können, lautet wie folgt:

            "skip": {
                "condition1": "value1",
                "condition2": 1300,
                ...etc
            }
Beachten Sie, dass numerische Werte nicht in Anführungszeichen eingeschlossen werden. Das macht nicht gelten für Größenwerte wie "640 x 360".

Beispiel

            "renditions": [
                {
                  "media_type": "video",
                  "reference_id": "ts1",
                  "format": "ts",
                  "type": "segmented",
                  "audio_codec": "aac",
                  "audio_bitrate": 96,
                  "video_bitrate": 1200.
                  "video_codec": "h264",
                  "keyframe_rate": 0.5,
                  "max_frame_rate": 30,
                  "width": 1280,
                  "height": 720,
                  "h264_profile": "main",
                  "skip": {
                      "min_video_bitrate": 1000,
                      "min_size": "640x360"
                    }
                },
                ... etc
            ]

Für das obige Profil (Fragment) wird die angezeigte Darstellung erstellt nur wenn das Quellvideo eine Videobitrate von mindestens 1000 kbps und eine Bildgröße von mindestens 640 x 360 Pixel hat.

Ein Aufnahmeprofil muss mindestens eine nicht überspringbare Wiedergabe einschließen. Wenn Sie versuchen, ein Profil zu erstellen, in dem jede Wiedergabe a skip -Objekt gibt die Ingest Profiles API einen Fehler zurück.

Felder überspringen

Unten sind die Felder (Bedingungen), die in der verwendet werden können skip Objekt:

Felder überspringen
Eigentum Typ Bezeichnung Ursprünglich
min_audio_bitrate ganze Zahl die minimale Audio-Bitrate, die die Quelle haben muss (kbps) Null
max_audio_bitrate ganze Zahl die maximale Audio-Bitrate, die die Quelle haben muss (kbps) Null
min_video_bitrate ganze Zahl die minimale Videobitrate, die die Quelle haben muss (kbps) Null
max_video_bitrate ganze Zahl die maximale Videobitrate, die die Quelle haben muss (kbps) Null
require_audio boolesch die Quelle muss eine Audiospur enthalten Null
require_video boolesch die Quelle muss eine Videospur enthalten Null
min_size String der Form "widthxheight" (zB "400x225") die Quelle muss mindestens diese Auflösung haben Null
max_size String der Form "widthxheight" (zB "1920x1080") die Quelle darf höchstens diese Auflösung haben Null
