Überblick

Dieses Beispiel und der zugehörige Code dienen als Leitfaden für Ihre Produktionsentwicklung. Der Code ist weder für den Produktionsgebrauch zertifiziert noch offiziell unterstützt.
Beachten Sie, dass diese App entwickelt wurde, um Brightcove-Kunden bei der Migration vom alten Smart Player zum neuen Brightcove Player zu helfen. Es wurde um Videos herum gebaut, die mit dem alten Aufnahmesystem hinzugefügt wurden, und gibt keine aussagekräftigen Ergebnisse für Dynamic Delivery-Videos zurück.

Dieses Tool generiert einen Bericht, der Folgendes enthält:

  • Zusammenfassung, die zeigt, wie viele Videos enthalten sind Nein HLS- oder MP4-Wiedergabe oder beides
  • Bericht für jedes Video mit HLS- und MP4-(HTTP)-Wiedergabe (mit Bitratenbereich und Bildgröße für jedes)

Ziel ist es, Sie bei der Migration vom Smart Player zum Brightcove Player zu unterstützen, indem Sie Videos identifizieren, die transkodiert werden müssen, um HLS- und/oder MP4-Wiedergaben zu erstellen.

Beachten Sie, dass Dynamic Delivery-Videos nicht überprüft werden, da dies nicht erforderlich ist.

Zugangsdaten abrufen

Wie die meisten RESTful-APIs von Brightcove ist die CMS-API erfordert OAuth-Zugriffstoken, um Anfragen zu autorisieren. Um Zugriffstoken zu erhalten, müssen Sie zunächst Client-Anmeldeinformationen mit Berechtigungen für die Arten von Vorgängen abrufen, die Sie anfordern möchten.

  1. Um Ihre Zugangsdaten zu erhalten, folgen Sie den Anweisungen in API-Anmeldeinformationen verwalten. Dies ist die empfohlene Methode, um Ihre Client-Anmeldeinformationen zu erhalten. Dieses Beispiel erfordert nur Video Read Berechtigungen, aber in den meisten Fällen möchten Sie wahrscheinlich alle Berechtigungen erhalten:

    Sie können Anmeldeinformationen auch mithilfe der OAuth-API abrufen, indem Sie die Anweisungen in einem der unten aufgeführten Anleitungen befolgen:

  • OAuth: Client-Anmeldeinformationen mit CURL abrufen
  • OAuth: Erhalten Sie Kundenanmeldeinformationen mit Postman

Wiedergabeinspektor

Mit der folgenden App können Sie einen Bericht zu den Wiedergabeversionen für alle Videos in Ihrem Konto erstellen. Videos ohne HLS- oder HTTP MP4-Wiedergabeversionen (progressiver Download) müssen neu transkodiert werden, damit sie im neuen Player gut funktionieren. Videos mit nur HLS-Wiedergaben mit niedriger Bitrate sollten ebenfalls neu transkodiert werden.

Siehe den Stift Inspektor für Videowiedergaben von Brightcove Learning Services (@rcroooks1969 ) An CodePen.

Proxy-Code

Um Ihre eigene Version der Beispiel-App auf dieser Seite zu erstellen, müssen Sie Ihren eigenen Proxy erstellen und hosten. (Die von Brightcove Learning Services verwendeten Proxys akzeptieren nur Anfragen von Brightcove-Domänen.) Ein Beispielproxy, das dem von uns verwendeten sehr ähnlich ist, jedoch ohne die Überprüfungen, die Anfragen von Nicht-Brightcove-Domains blockieren, finden Sie in dieses GitHub-Repository. Dort finden Sie auch grundlegende Anweisungen zur Verwendung und eine ausführlichere Anleitung zum Erstellen von Apps rund um den Proxy in Verwenden der REST-APIs.
