Primärfelder

Tabelle 1: Primäre Felder
Gebiet Typ Schreibgeschützt Standard Beschreibung
id Schnur Jawohl keiner die Playlist-ID
account_id Schnur Jawohl keiner die ID des Kontos
created_at ISO 8601 Datum-Uhrzeit-Zeichenfolge Jawohl keiner Datum-Uhrzeit-Wiedergabeliste wurde dem Konto hinzugefügt; Beispiel: "2014-12-09T06:07:11.877Z"
description Schnur Nein Null die kurze Beschreibung der Playlist - maximal 248 Single-Byte-Zeichen
favorite boolesch Nein falsch ob die Playlist in den Favoriten enthalten ist
limit Nummer Nein 100 Anzahl der Videos, die die Playlist enthalten kann (integer, 1 - 100) nur für Smart Playlists zurückgegeben
name Schnur Nein keiner Playlist-Titel - Pflichtfeld (maximale Länge: 248 Single-Byte-Zeichen)
reference_id Schnur Nein Null jeder Wert, der innerhalb des Kontos eindeutig ist
search Schnur Nein Null Suchstring zum Definieren der Videos für eine Smart Playlist (Beispiel: "+tags:"bird","birds"")
type Schnur Jawohl die Art der Wiedergabeliste Siehe Anmerkung[1] unter
updated_at ISO 8601 Datum-Uhrzeit-Zeichenfolge Jawohl keiner Datum-Uhrzeit-Wiedergabeliste wurde zuletzt geändert;[1-2] Beispiel: "2015-01-13T17:45:21.977Z"
video_ids Array Nein Array von IDs für die Videos in der Playlist Nur für manuelle Playlists zurückgegeben

Anmerkungen

  • [1]

    Die type Wert kann nur einer der folgenden sein: AUSDRÜCKLICH, ACTIVATED_OLDEST_TO_NEWEST , ACTIVATED_NEWEST_TO_OLDEST , ALPHABETICAL , PLAYS_TOTAL , PLAYS_TRAILING_WEEK , START_DATE_OLDEST_TO_NEWEST,START_DATE_NEWEST_TO_OLDEST

    EXPLICIT bedeutet, dass es sich um eine manuelle Wiedergabeliste handelt; alle anderen Typen sind intelligente Wiedergabelisten mit Namen, die ihrem Reihenfolgeschema entsprechen.
