    CMS API Fehlermeldung Referenz

    Dies ist ein Verweis auf Fehlermeldungen, die von der CMS-API zurückgegeben werden. Siehe auch die Fehlermeldungsreferenz für die Dynamic Ingest-API, die in Verbindung mit der CMS API für die Aufnahme von Videos verwendet wird.

    Allgemeine Fehlermeldungen

    Status Name Meldung Ursache
    400 BAD_REQUEST ILLEGAL_PATH: A request for more than 10 videos is not supported Die GET /videos/video_ids Basisanfrage kann nicht mehr als 10 Videos anfordern
    400 INVALID_SORT Attempted to sort by invalid property: '[property name]' Der sort Param wies auf ein ungültiges oder nicht existierendes Feld hin
    400 INVALID_SEARCH Search string was invalid Die Syntax der Suchzeichenfolge war ungültig, oder Sie haben die Suchzeichenfolge nicht UlenCode
    400 ILLEGAL_QUERY There was a problem with the query string Die Syntax der Suchzeichenfolge war ungültig - Beispiel: 1) eine tags Suche durchführen, die mit einem Komma endet oder ein ungeschlossenes Zitat hat
    400 BAD_VALUE Unrecognized field in the submitted data Rechtschreibfehler oder andere Verwendung eines nicht vorhandenen Feldes
    400 AD_CONFIG_NOT_FOUND Ad config not found Die in einer SSAI-Anfrage angegebene Anzeigenkonfiguration wurde nicht gefunden
    400 AD_CONFIG_INACTIVE Ad config is inactive Die in einer SSAI-Anfrage angegebene Anzeigenkonfiguration ist inaktiv
    400 REFERENCES_EXIST This video is referenced by at least one playlist. Sie versuchen, ein Video zu löschen, das in mindestens einer Wiedergabeliste enthalten ist.
    400 SHARED_VIDEO Delete of shared video failed. Das Löschen von geteilten Videos wird noch nicht unterstützt.
    401 UNAUTHORIZED Permission denied. Fehlendes oder ungültiges OAuth-Zugriffstoken. Wenn Sie ein Zugriffstoken bereitgestellt haben, hat es möglicherweise nicht den entsprechenden Umfang für diese Anfrage
    403 NOT_AVAILABLE Resource is not available. Die von Ihnen angeforderte Ressource ist vorübergehend nicht verfügbar - dies kann eine vorübergehende Bedingung sein, während eine Art der Verarbeitung des Videos durchgeführt wird. Wenn die Nachricht jedoch weiterhin besteht, wenden Sie sich an den Support.
    404 NOT_FOUND Resource does not exist. Sie haben eine Ressource angefordert, die nicht existiert - überprüfen Sie die Rechtschreibung.
    405 METHOD_NOT_ALLOWED The HTTP method used for the request is not allowed for this resource Die häufigste Ursache ist das Senden eines PUT, POST, PATCH oder DELETE an eine schreibgeschützte Ressource.
    406 NOT_ACCEPTABLE The HTTP Accept header has a value not allowed. Der Accept Header muss den Wert haben application/json ; andere Werte wie bewirken, dass die Anfrage mit diesem Fehler abgelehnt application/x-www-form-urlencoded wird.
    409 LIVE_VIDEOS_NOT_SHAREABLE This video cannot be shared because it is live. Aus ihnen erstellte Live-Streaming-Videos und Clips können nicht geteilt werden.
    409 REFERENCE_ID_IN_USE Reference id is already in use. Sie haben versucht, ein Video mit einer Referenz-ID zu erstellen, die bereits verwendet wird, oder einem Video, das bereits von einem anderen Video verwendet wird, eine Referenz-ID hinzuzufügen.
    409 CONCURRENT_UPDATE Update failed because of another update at the same time. Please try again. Entweder eine separate API-Anfrage, ein Studio-Benutzer oder ein Systemprozess aktualisiert das Video derzeit. Dies kann auch auftreten, wenn Sie schnell hintereinander API-Aktualisierungsanfragen für dasselbe Video asynchron stellen.
    409 CONFLICT Reference id is already in use. Um die Eindeutigkeit von Referenz-IDs zu gewährleisten, sperren Vorgänge mit Referenz-IDs die ID für bis zu 3 Minuten. Das heißt, wenn Sie eine Anfrage stellen, die eine Referenz-ID erstellt oder ändert, und dann eine andere Anfrage stellen, die diese Referenz-ID sofort verwendet, schlägt der Vorgang mit einem 409-Fehler fehl. Dies würde folgendes beinhalten: 1) Löschen eines Videos und anschließendes Verlangen, seine Referenz-ID einem anderen Video zuzuweisen; 2) versuchen, ein neues Video zu erstellen, das mit einem 503-Fehler (Dienst nicht verfügbar) fehlschlägt und dieselbe Anfrage erneut versucht.

    Bitte warten Sie mindestens 3 Minuten, nachdem Sie diesen Fehler erhalten haben, bevor Sie den Vorgang erneut versuchen. Beachten Sie auch, dass, wenn Sie versuchen, ein Video mit einer Referenz-ID zu erstellen, und der Versuch fehlschlägt (aus Gründen, die nicht mit der Referenz-ID zusammenhängen), diese Referenz-ID 30 Sekunden lang gesperrt wird und wiederholte Versuche, das Video zu erstellen, fehlschlagen, bis Sie lange genug darauf warten, dass die Referenz-ID entsperrt wird.
    409 CONFLICT Both accounts must be enabled for media sharing. Wenn ein Hauptkonto versucht, einem Kanal ein Affiliate-Konto für die gemeinsame Nutzung von Medien hinzuzufügen, müssen beide Konten für die Medienfreigabe aktiviert sein.
    409 UNSUPPORTED_MEDIA_TYPE   In der Anfrage fehlt höchstwahrscheinlich der Header: Content-Type: application/json.
    415 SHARING_DISABLED Master account is not enabled for media sharing. Dieser Fehler wird zurückgegeben, wenn Sie versuchen, ein Video von einem Konto zu teilen, für das die Medienfreigabe nicht aktiviert ist.
    422 ILLEGAL_FIELD Unrecognized field in the submitted data Rechtschreibfehler oder andere Verwendung eines nicht vorhandenen Feldes
    422 VALIDATION_ERROR (Die JSON-Daten waren nicht gültig - Fehlermeldungen variieren je nach Problem) Beispiele für Nachrichten:
    • Name: REQUIRED_FIELD (Erstellen Sie ein Video ohne Namen, aktualisieren Sie den Namen auf eine leere Zeichenfolge)
    • account_id: WRONG_ACCOUNT  (Konto-ID in URL stimmt nicht mit Konto-ID in JSON überein)
    • Ungültige Schlüssel: id (Die ID kann nicht geändert werden und sollte nicht als Schlüssel im JSON erscheinen)
    429 TOO_MANY_REQUESTS Too many requests Sie reichen zu viele gleichzeitige Anfragen oder zu viele Anfragen pro Sekunde ein
    500 UNKNOWN an unknown internal error occurred Problem im Brightcove System - versuchen Sie es später noch einmal.
    503 SERVICE_UNAVAILABLE The API is temporarily unavailable Backend-Problem - versuchen Sie es später noch einmal.
    504 TIMEOUT something took too long Server wahrscheinlich zu beschäftigt - versuchen Sie es später noch einmal.

    Fehler beim Teilen von Medien

    Fehler bei der Medienfreigabe werden nicht als separate Fehlerantwort auf die API-Anfrage zurückgegeben, sondern in einem error_message Feld in der normalen Antwort:

      [
  {
  "video_id" : "394872349182374",
  "affiliate_id" : "234987239487",
  "affiliate_video_id" : "30308254055202",
  "status" : "COMPLETE",
  "shared_at" : "2017-12-11T17:57:45.530Z",
  "updated_at" : "2017-12-11T18:03:32.789Z",
  "error_message" : "[{"error_code":"MISSING_CUSTOM_FIELDS","error_message":"Affiliate account is missing custom fields: [whisky]"}]"
  }
  ]

    Der error_message JSON wird ein Array von Fehlerobjekten sein, das Objekt hat error_code und error_message.

    error_code

    Der Fehlercode ist normalerweise spezifisch, aber es kann sich um einen der gültigen Fehlercodes handeln, die auch von einem PATCH- (Update-) Vorgang in einem Video zurückgegeben werden können. Die Freigabe-spezifischen Fehlercodes sind in der folgenden Tabelle aufgeführt.

    Fehlercodes für die Medienfreigabe
    error_code error_message Ursache
    MISSING_CUSTOM_FIELDS Affiliate account is missing custom fields: [field1, field2]} Der Master hat sich enforce_custom_fields auf eingestellt true , aber das Video wird geteilt referenzfelder, die nicht im Affiliate-Konto vorhanden sind.
    CONFLICT Affiliate account is not configured for geo restriction. Der Master hat enforce_geo auf true gesetzt und sein Konto ist für die Geo-Filterung konfiguriert, aber das Affiliate-Konto ist nicht für die Geo-Filterung konfiguriert.
    SHARING_DISABLED Affiliate account is not enabled for media sharing. Der Master hat versucht, ein Video mit einem Partner zu teilen, der nicht für die Medienfreigabe aktiviert ist.
    NOT_FOUND Resource does not exist. Etwas hat gefehlt. Normalerweise bedeutet dies, dass das Master-Video weg ist. Dies kann passieren, wenn auf eine Freigabeanfrage sofort ein Löschen des Master-Videos folgt. Dies kann auch bedeuten, dass das Hauptkonto noch keinen Kanal hat oder wenn das Affiliate-Konto nicht existiert oder kein gültiges Mitglied des Kanals ist. Dies kann passieren, wenn bei der Migration zu CMS API Media Sharing etwas schief gelaufen ist.
    UNKNOWN Sharing failed with an unknown error, error_id INSERT-UNIQUE-ERROR-ID Etwas Unerwartetes ist passiert. Die error_message wird eine Fehler-ID enthalten, die von DevOps verwendet werden kann, um die zugrunde liegende Ursache zu identifizieren. Ein erneuter Versuch des Aktienversuchs wird in vielen Fällen funktionieren.
    VALIDATION_ERROR {field}: ILLEGAL_VALUE Wenn {field} „Wirtschaft“ ist, besteht das Problem darin, dass das Hauptvideo „AD SUPPORT“ ist, für das Affiliate-Konto jedoch keine Anzeigen aktiviert sind. Tritt auch auf, wenn Sie die von den Videometadaten (eines nicht geteilten Videos) unterstützte Werbung auf Werbung einstellen, wenn das Konto für Anzeigen deaktiviert ist. Wenn {field} etwas anderes ist, ist eine Untersuchung erforderlich.
    REFERENCE_ID_IN_USE Reference id whatever-you-used is already in use. Die Referenz-ID war im Hauptkonto gültig, wird jedoch bereits vom Affiliate-Konto verwendet.
