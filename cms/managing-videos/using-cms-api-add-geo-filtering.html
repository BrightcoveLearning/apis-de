Einführung

Geofilterungseigenschaften können hinzugefügt werden, wenn Sie das Video zum ersten Mal mithilfe einer POST-Anfrage oder mithilfe einer PATCH-Anfrage erstellen, um die Videometadaten zu aktualisieren. Sehen Videos aktualisieren mit CMS-API ein Beispiel für eine Update-Anfrage.

Anstatt oder zusätzlich zur Geofilterung auf Videoebene können Sie Anwenden von Geofilterung auf Brightcove-Player über Richtlinienschlüssel. Video Cloud berücksichtigt sowohl die Richtlinien für die Geofilterung auf Video- als auch auf Player-Ebene und wendet beide an. Wenn ein Land durch eine der Richtlinien ausgeschlossen ist, ist das Video in diesem Land nicht verfügbar.

Beachten Sie, dass Ländercodes eingegeben werden sollten Kleinbuchstaben.

Einige Verlage haben die Möglichkeit beantragt, Videos sowohl auf US-Militärstützpunkte als auch auf die USA zu filtern. Um dies zu unterstützen, gibt es ein U.S. Militär Möglichkeit. Es gibt zwar keine hundertprozentig zuverlässige Möglichkeit, eine Militärbasis zu identifizieren, aber wenn eine IP gefunden wird, die sich in eine .mil-Domain auflöst und sich nicht in den USA befindet, wird sie als von einer US-Militärbasis stammend und nicht als von ihrem tatsächlichen Land stammend identifiziert Herkunfts. .mil-Adressen in den USA werden weiterhin als "uns" identifiziert.

Zugriff auf Geofilterung

Geofilterung ist ein kostenpflichtiges Add-on zu Video Cloud. Wenn Sie daran interessiert sind, das Geofilterungspaket zu erwerben, wenden Sie sich an Ihren Account Manager.

Genehmigung

Anfragen an die CMS-API benötigen einen Autorisierungs-Header, der Ihre Zugriffstoken enthält. Ausführliche Informationen zum Abrufen von Client-Anmeldeinformationen und deren Verwendung zum Abrufen von Zugriffstoken finden Sie im Übersicht über Brightcove OAuth.

Die Vorgänge, für die Sie Berechtigungen benötigen, um Video-Metadaten zu aktualisieren, sind video-cloud/video/update oder video-cloud/video/all.

Syntax

geo ist ein Objekt mit drei Eigenschaften:

Eigentum Typ Erforderlich Kommentare
geo.countries Array von Ländercode-Strings Nein Ein ISO 3166 Liste der 2-Buchstaben-Codes in Kleinbuchstaben für Länder, die zum Ansehen des Videos auf die weiße oder schwarze Liste gesetzt werden
geo.exclude_countries Boolesches Nein Wenn true , Ansehen ist nicht in Ländern erlaubt, die in die geo.countries aufführen; wenn false , Ansehen ist nur in diesen Ländern erlaubt.
geo.restricted Boolesches Nein Wenn true Geofilterung ist für Videos aktiviert

Anfrage

    https://cms.api.brightcove.com/v1/accounts/{account_id}/videos/{video_ID}

Anfragetext:

Der Anfragetext ist ein JSON-Objekt, das ein Array von Länderabkürzungen enthält, die eingeschlossen oder ausgeschlossen werden sollen:

    {"geo" : {
        "restricted" : true,
        "exclude_countries" : false,
        "countries" : ["ph","in","my","ja"]
      }
    }
