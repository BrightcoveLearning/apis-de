Einführung

Der Videodatenendpunkt bietet eine schnelle Möglichkeit, spezifische Daten für ein bestimmtes Video abzurufen. Aktuelle Anfragen geben die Gesamtaufrufe für das Video zurück. In Zukunft können zusätzliche Daten zurückgegeben werden. Der primäre Anwendungsfall für diesen Endpunkt besteht darin, Spielern auf stark frequentierten Seiten Informationen zu allen Zeitansichten bereitzustellen, auf denen große Mengen regelmäßiger Anfragen an die Analytics-API wäre zu langsam.

Endpunkt

Der Endpunkt für Videodatenanfragen ist:

    https://analytics.api.brightcove.com/v1/alltime/accounts/{account_id}/videos/{video_id}

Alle Anfragen sind GET.

Authentifizierung

Alle Anfragen an die Analytics-API werden über den OAuth-Dienst von Brightcove authentifiziert.

Die Authentifizierung basiert auf einem Zugriffstoken, das in einem Authorization-Header übergeben wird:

    Authorization: Bearer {your token}

Sie können Kundenanmeldeinformationen für die Analytics-API und verwenden Sie diese, um temporäre Zugriffstoken abzurufen - siehe die OAuth-Übersicht für eine Erläuterung der verschiedenen Ansätze, die Sie zum Abrufen von Client-Anmeldeinformationen und Zugriffstoken verwenden können, und API-Anmeldeinformationen verwalten Informationen zum Abrufen von Kundenanmeldeinformationen über Studio

Antwort

Die Antwort auf eine Anfrage ist JSON in der Form:

    {
        "alltime_video_views": 103123
    }

Mögliche Implementierung

So können Sie den Videodatenendpunkt verwenden, um Allzeitansichten für ein Video anzuzeigen, wenn es in einen Player geladen wird:

Hinweis: Dieser Endpunkt ist nur für Professional- und Enterprise-Konten verfügbar. Wenn Sie an einem Upgrade Ihres Kontos interessiert sind, wenden Sie sich bitte an Brightcove Sales oder Ihren Account Manager.
  1. Richten Sie das Caching ein, um Aufrufe nach Video-ID 5 bis 10 Minuten lang zu speichern.
  2. Erstellen Sie eine Logik auf Ihrem Server oder in Ihren Seiten, um zuerst den Cache zu überprüfen, um zu sehen, ob die Daten für die Alltime-Videoaufrufe für das aktuelle Video bereits vorhanden sind, und verwenden Sie sie, wenn dies der Fall ist.
  3. Wenn der Wert noch nicht im Cache gespeichert ist, stellen Sie eine Anforderung an den Videoinformationsendpunkt, um ihn abzurufen, und speichern Sie den Wert im Cache zusammen mit der Anzeige auf der Seite.
  4. Stellen Sie sicher, dass die Ausnahmebehandlung für den Fall eingerichtet wird, dass die Anforderung an den Videoinformationsendpunkt ein Timeout hat oder nicht erfolgreich ist. Möglicherweise werden die Videoaufrufe aller Zeiten einfach nicht angezeigt oder Sie verwenden den letzten Wert, den Sie haben.
