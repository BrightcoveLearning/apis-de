    Übersicht: Dimensionen, Felder und Parameter

    Dimensionen sind die wichtigsten Datenkategorien für Analytics API Datenberichte. Dieses Thema bietet eine interaktive Anleitung zu Dimensionen und den Feldern, die für sie zurückgegeben werden können. Außerdem wird angezeigt, welche Dimensionen in einem Bericht kombiniert werden können, und welche Felder für die verschiedenen Kombinationen verfügbar sind.

    Dimensionen und Felder

    Dimensionen sind die primären Datenmengen für Analysen. Um vollständige Leitlinien für die einzelnen Dimensionen anzuzeigen, klicken Sie in der Liste unten auf den Dimensionsnamen.

     
     

    Wählen Sie unten Dimensionen aus, um die Felder anzuzeigen, die dafür zurückgegeben werden können. Sie können auch auf die Schaltfläche Anfrage stellen klicken, um eine Beispielanfrage zu stellen und die Ergebnisse anzuzeigen. Wenn Sie mehrere inkompatible Dimensionen auswählen, wird eine entsprechende Meldung angezeigt.

    Hinweis: dass Sie das Feld nicht zurückgeben können video.custom_fields, aber in einigen Fällen können Sie zurückgeben video.custom_fields.{field_name}. Dies wird im Folgenden dargestellt, aber da es {field_name} sich nicht um einen echten Feldnamen handelt, wird er nicht in Beispielanfragen verwendet.

    Hinweise

    1. Standardmäßig video_view ist das einzige zurückgegebene Feld - andere Felder werden nur zurückgegeben, wenn sie im Wert des fields Parameters angegeben sind.
    2. Wenn Sie ein zurückzusendendes Feld angeben, das für die Dimensions- oder Dimensionskombination nicht unterstützt wird, wird ein UNSUPPORTED_FIELD_COMBINATION_ERROR Fehler zurückgegeben.
    3. Das bytes_delivered Feld enthält alle Daten, die Video Cloud den Kunden zur Verfügung stellt, einschließlich Videodaten, Bilder, Textspuren und andere Assets sowie den Player-Code selbst. Einige dieser Daten stammen von CDNs und sind möglicherweise bis zu 3 Tage lang nicht verfügbar.
    4. Zusätzlich zu den Feldern, die für die video Dimension angezeigt werden, können Sie auch video.custom_fields.{field_name}

    Beispiel anfragen

    Ein typischer Anwendungsfall für den Abrufen eines Berichts über mehrere Dimensionen: Sie möchten eine Aufschlüsselung der Videoaufrufe zwischen Desktop- und Mobilgeräten und möchten auch wissen, wie viele der Ansichten des Mobilgeräts auf iOS- gegenüber Android-Geräten waren und wie viele der Desktop-Ansichten auf Macs im Vergleich zu Windows-Computern waren. Derzeit gibt es im Studio Analytics Module keinen Standardbericht, der diese Informationen bereitstellt, aber Sie können ihn über diesen Analytics API Aufruf erhalten:

      https://analytics.api.brightcove.com/v1/data?accounts=57838016001&dimensions=video,device_type,device_os&from=2014-01-01&to=2014-04-01&fields=video_view

    (In diesem Fall bitten wir um Videoaufrufe für den Zeitraum vom 1. Januar bis zum 1. April im Jahr 2014.)

    Beispiel für die Verwendung von cURL

    Wenn Sie die API mit cURL ausprobieren möchten, finden Sie hier einige Anmerkungen:

    • Sie müssen zuerst ein Zugriffstokenerhalten
    • Da die URL für die Anfrage immer URL-Parameter enthält, müssen Sie sie in Anführungszeichen einschließen (einfach oder doppelt)

    Sample

    Hier ist ein Beispiel für cURL-Befehl:

      curl -s --header "Authorization: Bearer $ACCESS_TOKEN" \
  "https://analytics.api.brightcove.com/v1/data?accounts=$ACCOUNT_ID&dimensions=video&from=2017-04-04&limit=100"

    Wenn Sie ersetzen $ACCESS_TOKEN mit einem gültigen Zugriffstoken und $ACCOUNT_ID Mit Ihrer Konto-ID sollte diese Anfrage funktionieren. Beachten Sie, dass Sie diese Beispiel-App verwenden können, um ein Zugriffstoken zu generieren.

    Parameter

    Im Folgenden finden Sie eine Tabelle, die die im verfügbaren Parameter zusammenfasst Analytics API. Die Verwendung der Parameter wird in den folgenden Abschnitten ausführlicher besprochen.

    Parameter Erforderlich Beschreibung WERTE Default

    Konten

    Die Video Cloud-Konten, für die Sie einen Bericht benötigen, werden mit dem accounts Parameter angegeben. Beispiel:

      https://analytics.api.brightcove.com/v1/data?accounts={account1_id,account2_id}

    Woher filtert

    Die allgemeine Syntax für Filter lautet:

    where=dimension1==value1;dimension2==value2

    Beispiel:

    https://analytics.api.brightcove.com/v1?accounts=account_id(s)&dimensions=device_type&where=video==video_id;device_type==tablet

    Kommas werden als logische ORs und Semikolons als logische ANDs behandelt. Zum Beispiel where=video==1234,5678;player==9876 wird interpretiert als „wo Video = 1234 ODER 5678 UND Spieler = 9876"

    Leerzeichen und Sonderzeichen

    String-Werte sollten URI-codiert sein. Sie können Sonderzeichen auch mit einem „“ entkommen:

    where=search_terms==boston,%20ma

    Sie können jede Dimension als Filter verwenden, aber nur , wenn diese Dimension auch in der dimensionsangeforderten Dimension enthalten ist .

    Beachte Folgendes, wenn du nach Spielern filtrierst:

    • Brightcove Players - Geben Sie die Spieler-URL in diesem Format ein: players.brightcove.com/account_id/player_id _default
    • Wenn Sie eine Anwendungs-ID verwendet haben, um den Player zu identifizieren, geben Sie die Player-URL in diesem Format ein: players.brightcove.com/account_id/player_id_default/ application-id - zum Beispiel: players.brightcove.com/1752604059001/BJFYWrtUl_default/AlltimeViewsPlayerSample
    • Smart Players - Geben Sie die Spieler-ID ein

    Filtern nach Videoeigenschaften

    Mit dem where=video.q=={property}:{value} Spezialfilter können Sie Ihren Bericht auf eine bestimmte Gruppe von Videos beschränken, die auf einer Vielzahl von Eigenschaften basieren, darunter:

    • Tags
    • reference_id
    • custom_felder [1]
    • {a_specific_custom_field}
    • created_at

    Hinweise

    [ 1] Die grundlegende Syntax ist where=video.q==custom_fields:value (entspricht dem Wert in einem benutzerdefinierten Feld) oder where=video.q==myfield:value (entspricht dem Wert im spezifischen benutzerdefinierten Feld myfield). Wenn Sie nach bestimmten benutzerdefinierten Feldern suchen, beachten Sie, dass Sie nach dem internen Namen suchen müssen, nicht nach dem Anzeigenamen:

    Eine kurze Überprüfung, ob Sie den richtigen Namen verwenden: Der interne Name ist in Kleinbuchstaben geschrieben und enthält keine Leerzeichen.

    Beispiele

    Hier sind einige where Beispielfilter für die Suche nach Tags und benutzerdefinierten Feldern:

    Einzelne Tag
    where=video.q==tags:foo
    Mehrere Schlagworte:
    where=video.q==tags:foo,bar
    Benutzerdefinierte Felder
    where=video.q==custom_fields:foo
    Tags und benutzerdefinierte Felder
    where=video.q==tags:foo,bar+custom_fields:fish

    Eine vollständige Erklärung dieser Abfragesyntax finden Sie unter Using the CMS API: Suche nach Videos.

    Die mit diesem Filter abgerufenen Daten werden 24 Stunden lang zwischengespeichert und spiegeln daher keine Änderungen wider, die am vergangenen Tag vorgenommen wurden

    Datum-Bereiche

    Datumsbereiche, die in from und to Parameter für alle Arten von Berichten angegeben sind, können in verschiedenen Formaten angegeben werden:

    • Text-Werte:
      • to=now ( verfügbar und der Standardwert für alle Anfragen)
    • Epoch-Zeitwerte in Millisekunden, wie 1377047323000
    • Daten, die im internationalen Standarddatumsformat nach ISO 8601 ausgedrückt werden: YYYY-MM-DDFormat, wie 2013-09-12z. Für Daten, die in diesem Format ausgedrückt werden:
      • Jeder festgelegte Datumsbereich wird in der für das Konto festgelegten Zeitzoneinterpretiert
      • Die Zeit für das Datum wird als Mitternacht interpretiert ( 00:00:00 ) am angegebenen Datum in der für das Konto festgelegten Zeitzone
    • Relative Daten: Sie können eines der beiden ausdrücken to und from Werte relativ zum anderen in d (Tage) oder h (Stunden). Beispiel:
      • from=2015-01-01&to=31d
      • from=-48h&to=now
      • from=-2d&to=now ( wird die gleichen Ergebnisse wie im vorherigen Beispiel geben)
      • from=-365d&to=2014-12-31

      Beachten Sie, dass negative Zahlen (-2d) als „vor“ (der andere Wert) interpretiert werden und positive Zahlen (48 Stunden) als „von“ (der andere Wert) behandelt werden

    Um einen Bericht über eine Dimension wie „Video“ für einen einzelnen Tag zu generieren, legen Sie die Werte auf und von auf dieses Datum fest:

    ...&dimensions=video&from=2013-11-01&to=2013-11-01
    Hinweise:
    • Sie können den verfügbaren Datumsbereich für jeden Bericht mithilfe des Endpunkts abrufen /data/status - weitere Informationen finden Sie in der API-Referenz

    Begrenzen und Ausgleich

    Die limit ist die Anzahl der Artikel, die zurückgegeben werden sollen (Standard: 10). Um alle Artikel zurückzugeben, verwenden Sie limit=all. offset ist die Anzahl der zu überspringenden Elemente (Standard: 0). Sie können limit und offset zusammen verwenden, um eine App zu erstellen, die die Ergebnisse durchstößt.

    Abgesagte Daten

    Der reconciled Parameter ist ein boolescher Wert. Wenn dies so festgelegt ist true, sind die Ergebnisse auf abgesenkte Daten beschränkt. Wenn false, beschränken sich die Ergebnisse auf Daten in Echtzeit (stündlich nicht abgestimmt).

    Echtzeitdaten sind nur für die letzten 32 Tage verfügbar. Wenn Sie einstellen reconciled=false und die from Datum vor 32 Tagen, die API gibt keine Daten zurück, da zu diesem Zeitpunkt keine nicht abgestimmten Daten vorhanden sind.

    Geografische Berichte

    Dimensionen für geografische Analytik

    • country - Wie der Ländercode ISO-3611-1. zB: 'UNS'
    • region - Wie der ISO-3611-2-Regionscode. zB: 'US-WA'
    • city - Stadtname. zb: Seattle

    Hinweis: Für ein unbekanntes Land oder eine unbekannte Region gibt die API „ZZ“ als Code zurück (gemäß ISO-3611-alpha2).

    Felder und Sortierung

    Verwenden Sie den fields Parameter, um die Felder anzugeben, die Sie zurückgeben möchten. Standardmäßig video_view wird zurückgegeben und das Feld, das der Dimension entspricht, über die Sie berichten (z. B. destination_domain ), wird zurückgegeben. Siehe Dimensionen und Felder für weitere Details.

    Verwenden Sie den sort Parameter, um anzugeben, mit welchem Metrikfeld die zurückgegebenen Elemente sortiert werden, z. B.: sort=video_view. Sie können die Sortierreihenfolge umkehren, indem Sie das Sortierfeld negieren: sort= -video_view

    Die Felder, nach denen Sie zurückgeben oder sortieren können, hängen von den Metriken ab, die für die Dimension (en) zurückgegeben werden, über die Sie berichten. Diese Parameter sind streng validiert und Ihre Anfrage gibt einen Fehler zurück, wenn die fields oder sort Parameter zeigt auf ein Feld, das für die Anforderung nicht gültig ist.

    Für Videodimensionsberichte enthalten gültige Sortierwerte Videometadatenfelder: video.name , video.reference_id , und video.tags.

    video_name im Vergleich zu video.name

    • video_name ist der in Analytics-Daten aufgezeichnete Videoname
    • video.name ist der Videoname in den Video-Metadaten

    In den meisten Fällen sind die beiden Werte identisch, aber es gibt Fälle, in denen dies nicht der Fall ist:

    • In einigen Fällen sehen Sie möglicherweise einen Wert für video.name, aber nicht für video_name- in diesem Fall wurde der Videoname aus irgendeinem Grund nicht an den Analysedatensammelpunkt gesendet
    • In einigen Fällen können die Werte unterschiedlich sein - in diesem Fall wurde das Video irgendwann umbenannt: video.name ist der aktuelle Name des Videos in Ihrer Video Cloud-Medienbibliothek; video_name wird der Name des Videos des -Video zu dem Zeitpunkt, zu dem die Daten an den Analysedatensammelpunkt gesendet wurden (wenn Sie einen Zeitraum abfragen, in dem sich der Name geändert hat, ist der von der API zurückgegebene Wert der Name, der mit den meisten Ansichten während des Zeitraums verknüpft ist).

    Berechnete Felder

    Mithilfe der Syntax können Sie Ihren API-Anfragen berechnete Felder hinzufügen:

    fields=calulated_field_name:expression

    Sie können berechnete Felder verwenden, um Ihre eigenen benutzerdefinierten Felder aus vorhandenen Metriken zu erstellen oder ein vorhandenes Feld umzubenennen.

    Beachten Sie, dass Sie ein berechnetes Feld nicht als Teil eines Ausdrucks für ein anderes berechnetes Feld verwenden können.

    Der Name für das berechnete Feld kann eine beliebige URI-kompatible Zeichenfolge sein. Der Ausdruck kann reguläre Feldnamen und die folgenden arithmetischen Operatoren enthalten:

    • + ( zusätzlich)
    • - ( Subtraktion)
    • * ( Multiplikation)
    • / ( Abteilung)
    • ^ ( Exponent)
    • () ( Klammern)

    Beispiele

    fields=avg_seconds_viewed:video_seconds_viewed/video_view,video.name
    fields=avg_incomplete_ads:(ad_mode_begin-ad_mode_complete)/video_view,video.name
    fields=Video%20Views:video_view,video.name
