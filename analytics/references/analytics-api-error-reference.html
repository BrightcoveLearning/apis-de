Fehlermeldungen

HTTP-Code Fehlercode Beschreibung Auflösung
400 BAD_REQUEST Dieser Fehler weist darauf hin, dass die gestellte Anforderung einen clientseitigen Fehler aufwies. Die Nachricht enthält im Allgemeinen Informationen darüber, was der Fehler war. Die message Felder der Antwort enthalten Informationen darüber, was den Fehler verursacht hat, z. B. "ungültiger Wert für". sort Parameter.
400 UNSUPPORTED_FIELD_COMBINATION_ERROR Dieser Fehler weist darauf hin, dass die fields Parameter für einen Bericht enthalten ein oder mehrere Felder, die für den Berichtstyp nicht gültig sind. Die message Felder der Antwort enthält Informationen darüber, welche ungültigen Felder angegeben wurden.
400 TIME_RANGE_UNAVAILABLE Diese Antwort wird zurückgesendet, wenn das Verlaufs-/Batch-Verarbeitungssystem nicht über die von Ihnen angeforderten Daten verfügt.
401 UNAUTHORIZED Dieser Fehler weist darauf hin, dass die Autorisierung (OAuth2- oder LA-Token), die für die Anforderung verwendet wurde, nicht gültig ist. Überprüfen Sie das verwendete Autorisierungstoken. Es ist möglich, dass der verwendete Token das Zeitfenster überschreitet, für den er gültig war, und ein neuerer Token benötigt wird. Im Allgemeinen läuft jedes Token nach 5 Minuten ab. Das Autorisierungstoken kann als URL-Parameter angegeben werden access_token oder im Authorization Header. Beziehen auf API-Anmeldeinformationen verwalten für mehr Details.
404 NOT_FOUND Dieser Fehler wird im Allgemeinen zurückgesendet, wenn der angeforderte Endpunkt nicht existiert. Überprüfen Sie den Endpunkt, nach dem gefragt wird.
405 METHOD_NOT_ALLOWED Dieser Fehler tritt auf, wenn die API-Anfrage mit einer anderen HTTP-Methode als . erfolgt GET. Stellen Sie alle Anfragen an die API mit HTTP GET.
500 SERVER_ERROR Dieser Fehler tritt auf, wenn ein interner Fehler auf dem Server vorliegt. Der Server sendet diese möglicherweise zurück, wenn sie nicht verfügbar ist oder ein vorübergehendes Problem auftritt. Versuchen Sie es in einiger Zeit mit einem gewissen exponentiellen Backoff erneut. Wenn das Problem weiterhin besteht, erstellen Sie bitte ein Support-Ticket.
503 PROCESSING Das bedeutet, dass der Server die Anfrage erhalten hat und sie gerade bearbeitet. Die Analytics-API kann diese Nachricht zurücksenden, wenn eine Abfrage mit langer Laufzeit auftritt. Sobald die Abfrage abgeschlossen ist, wird sie für bis zu 5 Minuten im Cache des Servers gespeichert. Daher empfehlen wir, die API 4 Minuten nach Erhalt dieses Fehlers abzufragen
