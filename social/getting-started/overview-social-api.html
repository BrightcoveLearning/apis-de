Einführung

Die Social API ermöglicht es, den Social-Sharing-Status und den Verlauf für Ihre Videos abzurufen. Es soll hauptsächlich bei CMS-Integrationen helfen, damit Ihre Benutzer diese Informationen in Ihrem eigenen CMS-System anzeigen können.

Ausführliche Informationen zu Anfragen finden Sie im API-Referenz.

Basis-URL

Die Basis-URL für die API lautet:

  https://edge.social.api.brightcove.com/v1

Authentifizierung

Die Authentifizierung für Anfragen erfordert eine Autorisierungs-Header :

        Authorization: Bearer {access_token}

Die access_token ist ein temporäres OAuth2-Zugriffstoken, das von der Brightcove-OAuth Service. Ausführliche Informationen zum Abrufen von Client-Anmeldeinformationen und deren Verwendung zum Abrufen von Zugriffstoken finden Sie im Übersicht über Brightcove OAuth Thema.

Betrieb

Wenn Sie Client-Anmeldeinformationen anfordern, müssen Sie den gewünschten Kontozugriffstyp oder die gewünschten Vorgänge angeben. Im Folgenden finden Sie eine Liste der derzeit unterstützten Operationen für die Soziale API :

  • video-cloud/social/status/read

Beachten Sie, dass die Social API zwar in der Beta-Phase ist, aber Berechtigungen dafür sind nicht verfügbar im Abschnitt Studio-Admin zum Verwalten von API-Anmeldeinformationen. Stattdessen müssen Sie die OAuth-API verwenden, um Ihre Anmeldeinformationen zu erstellen. Brightcove Learning Services hat jedoch ein einfaches Online-Tool dafür.

Zusammenfassung der Anfragen

Nur in der aktuellen Version der API GET Anfragen werden unterstützt. Es gibt zwei Gruppen von Anfragen, die in den folgenden Abschnitten zusammengefasst werden.

Statusanfragen

Statusanfragen gibt den aktuellen Social-Sharing-Status für Videos zurück. Sie können den Status aller Videos in Ihrem Konto oder eines bestimmten Videos abrufen.

Verlaufsanfragen

Verlaufsanfragen den Social-Sharing-Verlauf für Videos zurückgeben. Sie können den Verlauf für alle Videos in Ihrem Konto oder für ein bestimmtes Video abrufen.
