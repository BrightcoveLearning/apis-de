Einführung

Mit der Social API können Sie den Status und den Verlauf der Freigabe von Videos auf Social-Media-Plattformen abrufen. Mit dieser Schnellstartanleitung können Sie die verfügbaren Anfragen und deren Antworten anzeigen, indem Sie entweder Ihr eigenes Konto oder ein Brightcove Learning Services-Beispielkonto verwenden.

Hinweise zur Beispiel-App

  • Dies ist ein Schnellstart, der Ihnen ein Modell zum Erstellen einer App rund um die Social API geben soll. Es ermöglicht Ihnen, grundlegende Anforderungen zu testen, verwendet jedoch keinen der Parameter, die diese Anforderungen verwenden können. Um diese zu sehen, schauen Sie auf die API-Referenz.
  • Die App stellt auch einige Anfragen an die CMS-API um die Namen der Videos abzurufen, die von der ursprünglichen Anfrage nach dem Status des sozialen Teilens von Videos zurückgegeben wurden. Dies dient nur dazu, die Videoauswahl für die anderen Anfragen zu füllen, da die Social API nur Video-IDs und nicht ihre Namen zurückgibt.
Dieses Beispiel und der zugehörige Code dienen als Leitfaden für Ihre Produktionsentwicklung. Der Code ist weder für den Produktionsgebrauch zertifiziert noch offiziell unterstützt.

Proxy-Code

Um Ihre eigene Version der Beispiel-App auf dieser Seite zu erstellen, müssen Sie Ihren eigenen Proxy erstellen und hosten. (Die von Brightcove Learning Services verwendeten Proxys akzeptieren nur Anfragen von Brightcove-Domänen.) Ein Beispielproxy, das dem von uns verwendeten sehr ähnlich ist, jedoch ohne die Überprüfungen, die Anfragen von Nicht-Brightcove-Domains blockieren, finden Sie in dieses GitHub-Repository. Dort finden Sie auch grundlegende Anweisungen zur Verwendung und eine ausführlichere Anleitung zum Erstellen von Apps rund um den Proxy in Verwenden der REST-APIs.
