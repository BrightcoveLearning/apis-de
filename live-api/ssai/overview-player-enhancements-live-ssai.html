Einführung

Mit serverseitigen Anzeigen können Sie den Anzeigenumsatz maximieren, indem Sie hochwertige Anzeigen als nahtlosen Livestream in Ihre Inhalte einfügen. Da Anzeigen serverseitig eingebunden werden, können Benutzer keine Werbeblocker nutzen und eine Implementierung funktioniert über mehrere Plattformen und Geräte hinweg. Sie müssen nicht für jedes Gerät benutzerdefinierte Clients erstellen.

Die Player Enhancements for Live SSAI von Brightcove bauen auf serverseitigen Anzeigen auf, indem sie die Benutzererfahrung erweitern und Klickeinnahmen erzielen. Mit diesem Toolkit können Sie Livestreams clientseitige Komponenten hinzufügen, darunter Anzeigenzählungen, Anzeigen-Countdown-Timer, Click-through-Anzeigen und Companion-Anzeigen.

Das Toolkit bietet auch eine weitere Möglichkeit, Tracking-Ereignisse auszulösen. Tracking-Daten werden dem Client angezeigt, sodass Sie die Flexibilität haben, Tracking-Ereignisse entweder vom Client oder vom Server auszulösen. Beachten Sie, dass Werbeblocker das clientseitige Tracking verhindern können.

Derzeit unterstützt diese Funktion keine überspringbaren Anzeigen.

Kontoeinrichtung

Wenden Sie sich an Ihren Account Manager, um Ihr Konto für Live-Streams, Server-Side Ad Insertion (SSAI) und Player-Erweiterungen für Live-SSAI aktivieren zu lassen.

Spezifikationen

Die Player-Verbesserungen von Brightcove für Live-SSAI verbessern die Endbenutzererfahrung durch die Bereitstellung der folgenden Funktionen:

Zeigt dem Client Tracking-Ereignisse an

Tracking-Daten, einschließlich Impressionen, Quartile, CreativeView, Pause, Stummschaltung, werden dem Client angezeigt. Dies gibt Ihnen die Flexibilität, Tracking-Ereignisse entweder vom Client oder vom Server auszulösen. Clientspezifische Ereignisse wie Click, Pause und Mute sollten immer vom Client ausgelöst werden.

Die ExtendedMetadata aus Ihren VAST-Antworten ist anpassbar und für den Kunden enthalten.

Werbeunterbrechungen, die im DVR-Fenster eines Streams geliefert werden, folgen denselben Aktionen.

Bietet Informationen zu Werbeunterbrechungen

Die Dauer der Werbeunterbrechung hilft, die Zuschauer mit dem Stream zu beschäftigen.

Informationen zu Werbeunterbrechungen werden an den Client weitergegeben. Jede Anzeigendauer innerhalb einer Pause wird dem Kunden angezeigt. Der Client bietet einen konfigurierbaren Countdown für die Pause oder die Anzahl der Anzeigen in der Pause sowie einen Countdown-Timer für jede.

Aktiviert Click-through-Anzeigen

Mit Click-through-Anzeigen können Sie den Umsatz maximieren und den Anzeigenverkauf aufeinander abstimmen.

Klick-URLs vom Ad-Server werden an den Wiedergabe-Client übergeben. Das Video ist anklickbar und sendet Tracking-Ereignisse beim Klicken des Benutzers. Wenn darauf geklickt wird, hält der Client den Stream an und öffnet das Ziel in einem neuen Browser-Tab oder einer separaten Webansicht.

Da Fernseher keinen Webbrowser unterstützen, sind Click-through-Anzeigen auf verbundenen Fernsehern nicht verfügbar.

Aktiviert Companion-Anzeigen

Companion-Anzeigen bieten ein weiteres Format zum Anzeigen von Anzeigeninhalten. Dabei handelt es sich in der Regel um Text, Display-Anzeigen oder Rich Media, die außerhalb des Players erscheinen. Der Vorteil dieser Anzeigen besteht darin, dass sie während der Videowiedergabe sichtbar bleiben können.

Companion-Anzeigen erscheinen nicht auf verbundenen Fernsehern.

Einschränkungen

Derzeit ist diese Funktion nur für die folgenden Regionen verfügbar:

  • ap-südöstlich-1 (Singapur)
  • ap-südöstlich-2 (Sydney)
  • ap-Nordost-1 (Tokio)
  • eu-central-1 (Frankfurt)
  • us-west-2 (Oregon)
