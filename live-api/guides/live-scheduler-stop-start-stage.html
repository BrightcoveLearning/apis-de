Anforderungen

  • Der Live-API-Schlüssel für Ihr Konto.
  • In Ihrem Live-Konto muss die Planerüberschreibung aktiviert sein.
  • Ein bestehender Live Job mit "statisch": wahr (sehen Statische Einstiegspunkt-Jobs für mehr Informationen).
Beachten Sie, dass Sie die Aktivierung und Deaktivierung auch im Live-Modul planen können - siehe Automatischer Start/Stopp von Live-Events im Kontrollraum für Details

Erstellen Sie einen Planer

Nehmen wir an, wir möchten eine 2-stündige Veranstaltung planen, die um beginnt 2021-04-02T14:00:00+00:00.

Zeitstempel müssen im UTC-Format als Unix-Zeitstempel in Sekunden formatiert sein. Sie können verwenden https://www.unixtimestamp.com/ zur Konvertierung.

Beachten Sie, dass die beschriebenen Aktivierungszeitbeschränkungen Hier gelten weiterhin für geplante Aktivierungen, daher empfehlen wir, eine Zeit einzuplanen, die mindestens 60 Sekunden vor der gewünschten Startzeit liegt.

Für dieses Beispiel wählen wir 5 Minuten vor der beabsichtigten Startzeit:

Anfang: 2021-04-02T13:55:00+00:00 = 1617386100 Stopp: 2021-04-02T16:00:00+00:00 = 1617393600

Um den Scheduler zu implementieren, machen Sie Folgendes: POST Anfrage:

https://api.bcovlive.io/v1/scheduler/jobstartstop

Überschriften

Content-Type: application/json
		X-API-KEY: {your API Key}

Anfragetext: einzelner SEP-Job

{
			"job_id": "your_Job_ID",
			"activation": {
				"time_utc": 1617386100
			},
			"deactivation": {
				"time_utc": 1617393600
			},
			"description": "My Wednesday broadcast"
		}

Beispielantwort

 
{
	"account_id": "1752604059001",
	"activation": {
		"state": "pending",
		"time_utc": 1617386100
	},

Anforderungstext: redundante Gruppe von SEP-Jobs

Bei redundanter Gruppenaktivierung besteht der einzige Unterschied darin, dass die redundant_group_id wird verwendet für die job_id , und der redundant_group Flag ist enthalten (auf gesetzt true).

{
			"job_id": "your_Redundant_Group_ID",
			"activation": {
				"time_utc": 1617386100
			},
			"deactivation": {
				"time_utc": 1617393600
			},
			"description": "My Wednesday broadcast",
      "redundant_group": true
		}

Aktualisieren Sie einen Planer

Sie können die aktualisieren Aktivierung und/oder Deaktivierung Zeit für einen Scheduler, vorausgesetzt, der Job ist in a steht aus Zustand ("zustand": "ausstehend"). Senden Sie dazu eine STELLEN Anfrage zu:

https://api.bcovlive.io/v1/scheduler/jobstartstop/{workflow_id}

Die Überschriften sind die gleichen wie für die POST oben gezeigte Anfrage.

Anfragetext

Fügen Sie im Anfragetextobjekt nur die Aktivierung und/oder Deaktivierung Felder.

Um das Ereignis beispielsweise 30 Minuten früher als geplant zu beenden:

Anfragetext

{
	"deactivation": {
		"time_utc": 1617391800
	}
}

Beispielantwort

 
{
	"account_id": "1752604059001",
	"activation": {
		"state": "pending",
		"time_utc": 1617386100
	},
	"deactivation": {
		"state": "pending",
		"time_utc": 1617391800
	},
	"description": "My Wednesday broadcast",
	"job_id": "15981d93877e490185e6ebc9af2c79a3",
	"type": "jobstartstop",
	"workflow_id": "d44cf29f4f184757ac0995beb8a0097e"
}

Einen Terminplaner stornieren

Sie können die Aktivierung und Deaktivierung mit a abbrechen LÖSCHEN Anfrage:

https://api.bcovlive.io/v1/scheduler/jobstartstop/{workflow_id}

Sie können dies auch nach der Aktivierung tun, um die Deaktivierung einfach abzubrechen.

Überschriften

Die einzige erforderliche Kopfzeile ist:

X-API-KEY: your API Key

Anfragetext

Es gibt keinen Anfragetext für diese Anfrage.

Beispielantwort

 
{
		"account_id": "1752604059001",
		"activation": {
			"state": "cancelled",
			"time_utc": 1617386100
		},
		"deactivation": {
			"state": "cancelled",
			"time_utc": 1617391800
		},
		"description": "My Wednesday broadcast",
		"job_id": "15981d93877e490185e6ebc9af2c79a3",
		"type": "jobstartstop",
		"workflow_id": "d44cf29f4f184757ac0995beb8a0097e"
	}

Benachrichtigungen

Beim Erstellen des Workflows können Sie optional Benachrichtigungen konfigurieren. Sie müssen eine URL für unseren Service angeben POST zu. Wir senden eine Benachrichtigung, wenn der Scheduler die Aktivierungs- und Deaktivierungsaufrufe durchführt. Sie können auch so konfigurieren, dass Sie eine Benachrichtigung erhalten n Sekunden vor der Aktivierung und/oder Deaktivierung. Siehe die Live-Planer-Benachrichtigungen für mehr Details.

Beispielanfragetext

 
{
		"job_id": "your Job ID",
		"activation": {
			"time_utc": 1612274100,
			"notification": 600
		},
		"deactivation": {
			"time_utc": 1612281600,
			"notification": 600
		},
		"description": "My Tuesday broadcast",
		"notification_url": "http://example.com/live/jobstartstop/callbacks"
	}
