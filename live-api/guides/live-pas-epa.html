PAS und EPA befinden sich derzeit in der Version mit eingeschränkter Verfügbarkeit. Wenn Sie an der Nutzung dieser Dienste interessiert sind, wenden Sie sich an Ihren Account Manager.

Einführung

Für einen Live-Job kann entweder PAS oder EPA aktiviert werden, um den von DRM bereitgestellten Schutz zu erweitern. PAS bietet eine begrenzte Anzahl von Einschränkungen: Begrenzung der Wiedergabe auf einen bestimmten Benutzeragenten oder Begrenzung der Anzahl der IP-Adressänderungen oder der Gesamtnutzung. Im Live-Fall ermöglicht es einem Kunden auch, die DRM-CRT für diesen Benutzer zu überschreiben. Diese Einschränkungen sind direkt in der Wiedergabeberechtigung angegeben JWT.

EPA bietet viel umfassendere Beschränkungen. Ein Satz dieser Einschränkungen wird über die Playback Rights Management API konfiguriert und mit einem Wiedergaberecht bezeichnet id. Dies id kann mit dem Video verknüpft werden (mit dem playback_rights_id unten gezeigtes Attribut) oder in den JWT. Der zweite Satz umfasst Einschränkungen auf Benutzerebene, wie z. B. Beschränkungen für die Parallelität und die Anzahl der Geräte. Diese werden direkt in den JWT-Ansprüchen angegeben (die auch eine Benutzer-ID enthalten müssen [vuid Anspruch]).

Für beide Modi muss die Jobanfrage Folgendes enthalten: videocloud.video -Attribut, damit ein Video Cloud-Video erstellt und mit dem Live-Job verknüpft wird. Die playback_rights_id -Attribut kann im Video Cloud-Video aktualisiert werden, um die Wiedergabebeschränkungen über das CMS-API.

Sie müssen auch ein JWT erstellen mit Ansprüchen, die PAS oder EPA entsprechen, je nachdem, welche Sie verwenden. Das JWT muss in alle Lizenzserveranforderungen eingeschlossen werden. Wenn Sie einen eigenen Player implementieren, müssen Sie diesen Wert entweder in a BCOV-Auth Kopfzeile oder bcov_auth Abfrageparameter (zusätzlich zum DRM-Token).

Erstellen Sie einen Live-Job mit EPA/PAS

Die folgenden neuen Felder werden verwendet, um EPA und/oder PAS zu einem Live-Job hinzuzufügen:

Neue Felder für die drm Objekt:

require_playback_token(boolesch)
Aktiviert PAS. Playback Auth JWT wird benötigt, wenn eine Lizenz angefordert wird.
check_playback_rights(boolesch)
Aktiviert EPA. Playback Auth JWT wird benötigt, wenn eine Lizenz angefordert wird. Verwendet die mit Video oder JWT verknüpfte Wiedergaberechte-ID, um Einschränkungen durchzusetzen.

Neues Feld für die videocloud.video Objekt:

Beachten Sie, dass dieses Feld mithilfe der CMS-API zum Video hinzugefügt werden muss nach Sie erstellen den Live-Job, da er beim Erstellen des Videos nicht hinzugefügt werden kann.
playback_rights_id(Zeichenfolge)
Verknüpft bestimmte EPA-Wiedergaberechte mit Video.

CMS-API-Anfrage

Um dem Video die Wiedergaberechte-ID hinzuzufügen, senden Sie eine Video aktualisieren Anfrage an die CMS-API mit dem folgenden Anfragetext:

{
	"playback_rights_id": "your playback_rights_id"
}

Player-Implementierung

Wenn Sie den Brightcove-Webplayer oder eines der nativen Brightcove-Player-SDKs zum Bereitstellen des Livestreams verwenden, ist die Implementierung für PAS oder EPA dieselbe wie für VOD. Siehe einen der folgenden Punkte:

PAS

  • Verwenden von DRM für die Wiedergabeautorisierung mit dem Brightcove Player
  • Verwenden von Playback Authorization DRM mit den nativen SDKs

EPA

  • Verwenden von Wiedergaberechten mit Brightcove Player
  • Verwenden von Wiedergaberechten mit den nativen SDKs

verwandte Themen

  • PAS
  • EPA
  • Erstellen von JSON-Webtoken (JWT)
  • Verwenden von DRM für die Wiedergabeautorisierung mit dem Brightcove Player
  • Verwenden von Playback Authorization DRM mit den nativen SDKs
  • a href="https://apis.support.brightcove.com/playback-rights/guides/using-epa-with-bcp.html">Verwenden von Wiedergaberechten mit Brightcove Player
  • Verwenden von Wiedergaberechten mit den nativen SDKs
