Anforderungen

  • Ein für die Live-API eingerichtetes Konto mit aktivierter mehrsprachiger Audiofunktion (für Details wenden Sie sich an Ihren Account Manager)
  • Ein API-Schlüssel für die Live-API (wird Ihnen bei der Einrichtung des Kontos zur Verfügung gestellt)
  • Ein Encoder, der das RTP-Protokoll unterstützt. Mehrsprachige Audiounterstützung ist beschränkt auf rtp oder rtp-fec Protokoll.
  • Die PID jeder Audiospur muss beim Erstellen des Jobs bekannt sein.

Erstellen Sie Ihren Live-Job

Um Ihrem Job mehrere Audiotracks hinzuzufügen, muss beim Erstellen eines Live-Jobs eine Liste von Audiotracks im Anfragetext bereitgestellt werden:

{
  "live_stream": true,
  "region": "us-west-2",
  "protocol": "rtp",
  "cidr_whitelist": [ /* omitted… */ ],
  "outputs": [ /* omitted… */ ],
  "alternate_audio": {
    "tracks": [ { Track Details } ]
  }
}

Unterstützte Felder für die Track Objekt

Die folgende Tabelle enthält eine vollständige Beschreibung der unterstützten Felder im track Objekt

Track-Felder
Gebiet Beschreibung
pid Erforderlich. Die PID, die aus dem eingehenden TS für den Track ausgewählt werden soll, der in der an Wowza gesendeten Stream-Datei angegeben ist.
language Erforderlich. Der Code für die zu verwendende Sprache. Derzeit ist dies flexibel und kann in jedem der verschiedenen Sprachformate erfolgen, die von den Spezifikationen erforderlich/unterstützt werden, insbesondere RFC5456/BCP47, das gemäß den Anforderungen auch ISO-639 abdeckt. https://tools.ietf.org/html/rfc5646

HLS > LANGUAGE Gebiet

DASH > AdaptationSet:lang Gebiet
variant Optional. Zuordnung zu den DASH-Rollenschemawerten: main, alternative, commentary, Supplementary, Dub. Wird in der DASH-Rolle verwendet urn:mpeg:dash:role:2011. Standardmäßig auf main.

HLS > EIGENSCHAFTEN siehe https://tools.ietf.org/html/rfc8216#section-4.3.4.1
label Optional. Eine für den Track zu verwendende Textbeschreibung, die nach Möglichkeit vom Spieler bevorzugt verwendet wird. Benutzt in: HLS > NAME Gebiet. Standardeinstellungen für die language mit einer ID für jeden Track zB en-0 wenn nicht angegeben.
streams[] Optional. Wenn keine Kopie vorhanden ist, verwenden Sie nur die Standard-Audioeinstellungen.

HLS > GROUP-ID. (Objekt)

Beispiel: { “source”: “720p” } - Übereinstimmung mit dem Label der Ausgabequelle.

Dies imitiert die streams Feld des Ausgabetyps der Wiedergabeliste.
default Erforderlich, außer bei Standardeinstellung. Welche Audiospur soll markiert werden als DEFAULT in dem EXT-X-MEDIA sowie welche in den Videostream gemuxt werden sollen *

Wenn nicht vorhanden, ist die erste Standardeinstellung.

* Wenn mehrere Playlists mit unterschiedlichen Standardeinstellungen definiert sind, ist der Standardtrack der obersten Ebene der gemuxte

Konfigurieren Sie Ihren Encoder

Konfigurieren Sie Ihren Encoder so, dass er MPEG-TS über RTP an die stream_url aus der Create-Antwort liefert. Stellen Sie sicher, dass die im Anfragetext bereitgestellten Audiospuren mit der richtigen PID in Ihrem Encoder konfiguriert sind.

Testwiedergabe

Laden Sie die play_url in die Brightcove-Player oder Safari, um die Wiedergabe zu bestätigen.

DRM_Wiedergabe

Derzeit wird DRM nur über den Catch-up-Dienst (Timeshift) unterstützt. Fügen Sie diese Felder zu Ihrem Anfragetext für Job erstellen hinzu:

{
  "drm": {
    "modes": [
      "all"
    ]
  },
  "timeshift": {
    "type": "single",
    "startover_duration": 36000,
    "skip_live_output": true
  }
}

Die Antwort umfasst DRM-Ausgaben und a timeshift_url für die Wiedergabe zu verwenden.
