Einführung

DRM kann zu Live-Jobs hinzugefügt werden, indem die DRM -Objekt im Anforderungstext, wenn Sie den Job erstellen. Die DRM Objekt hat zwei Eigenschaften:

  • modes auf ein Array oder einen oder mehrere der folgenden Werte setzen: "all", "fairplay", "playready", "widevine", "widevine:dash" oder "widevine:hls"
  • token_expires_in auf eine Zahl zwischen 43200 (Standard) und 31622400 (die Anzahl der Sekunden vor Ablauf des Tokens) eingestellt

Einstellung mode to all führt dazu, dass alle verfügbaren DRM-Typen auf die Ausgänge angewendet werden. Standardmäßig wird DRM wie folgt auf die Ausgaben angewendet:

  • FairPlay: auf HLS-Ausgänge angewendet
  • Playready: auf DASH-Ausgänge angewendet
  • Widevine: auf HLS-Ausgänge angewendet

Widevine kann auf DASH . angewendet werden Ausgänge statt HLS. Um Widevine mit DASH als Standard zu machen, ist eine Überschreibung auf Kontoebene erforderlich. Wenden Sie sich an den Support, wenn diese Überschreibung auf Ihr Konto angewendet werden soll. Im nächsten Abschnitt sehen Sie, wie Sie angeben können widevine:hls oder widevine:dash für bestimmte Jobs, unabhängig von der Standardeinstellung für Ihr Konto. Beachten Sie, dass die API-Antwort (wie die unten gezeigte) je nach Konfiguration Ihres Kontos/Jobs entweder widevine:hls oder widevine:dash Ausgänge, aber nicht beides.

Beachten Sie, dass in Ihrem Servicevertrag mit Microsoft Parallelitätsbeschränkungen für PlayReady definiert sind. Sehen dieses Microsoft-Dokument für Informationen zur Verwaltung der Parallelität. Wenn das Gleichzeitigkeitslimit für ein Video überschritten wird, kann es für Zuschauer über die maximale Anzahl hinaus nicht wiedergegeben werden.

Erstellen Sie Live-Jobbeispiele

Alle DRM-Modi

Hier ist ein Beispiel für einen Anforderungstext zum Verpacken von Ausgaben mit allen verfügbaren DRM-Typen:

  {
    "live_stream": true,
    "region": "us-west-2",
    "reconnect_time": 300,
    "ad_insertion": false,
    "static": false,
    "drm":{"modes":["all"]},
    "outputs": [{
        "label": "hls720p",
        "live_stream": true,
        "height": 720,
        "video_bitrate": 2000,
        "segment_seconds": 6,
        "_keyframe_interval": 60
      },
      {
        "label": "hls480p",
        "live_stream": true,
        "height": 480,
        "video_bitrate": 1000,
        "segment_seconds": 6,
        "_keyframe_interval": 60
      }]
  }

Job bei Widevine-HLS

So erstellen Sie Widevine-verpacktes DASH Ausgaben (ohne eine Kontoüberschreibung zu erhalten, um dies zum Standard zu machen), spezifizieren Sie widevine:dash in dem modes wie im Beispiel unten gezeigt. Wenn Sie Ihr Konto standardmäßig so eingestellt haben, dass Widevine mit DASH verwendet wird, können Sie Widevine-HLS-Ausgaben für einen bestimmten Job erhalten, indem Sie Folgendes angeben: widevine:HLS in dem modes.

Musteranfrage

  {
    "live_stream": true,
    "region": "us-west-2",
    "reconnect_time": 300,
    "ad_insertion": false,
    "static": false,
    "drm":{"modes":["fairplay","widevine:dash"]},
    "outputs": [{
        "label": "hls720p",
        "live_stream": true,
        "height": 720,
        "video_bitrate": 2000,
        "segment_seconds": 6,
        "_keyframe_interval": 60
   },
   {
        "label": "hls480p",
        "live_stream": true,
        "height": 480,
        "video_bitrate": 1000,
        "segment_seconds": 6,
        "_keyframe_interval": 60
   }]
  }
