Erstellen Sie einen Clip-Planer-Workflow

Beim Erstellen eines Clips müssen Sie sowohl die Clip-Zeitgrenze (im Ausgang Feld) und eine geplante Zeit (time_utc Feld), um die Clip-Anfrage zu stellen.

Die Ausgang -Feld im Anforderungstext unterstützt alle die gleichen Felder des Ausgänge Feld im Standard-VOD-Endpunkt. Jedoch, geplante Clips unterstützen nur eine einzelne Ausgabe und keine Liste. Sehen Support-Dokumente für VOD-Clipping für Details zu verfügbaren Feldern.

Normalerweise würden Sie einstellen time_utc um die Endbegrenzung Ihres Clips anzupassen. Die geplante Zeit MUSS hinter der Endgrenze des Clips liegen, andernfalls schlägt die Clipanforderung zum geplanten Zeitpunkt fehl.

Zeitstempel für die time_utc Das Feld muss im UTC-Format als Unix-Zeitstempel in Sekunden formatiert sein. Sie können verwenden https://www.unixtimestamp.com/ zur Konvertierung.

Nehmen wir an, wir möchten einen 15-minütigen Clip planen, der um beginnt 2021-02-09T13:55:00+00:00. Wir können die nutzen output.start_time und output.end_time um unsere Clipgrenze als Unix-Zeit in Sekunden anzugeben. (Wieder, siehe Support-Dokumente für VOD-Clipping für Details zu verfügbaren Feldern).

Startzeit: 2021-02-09T13:55:00+00:00 = 1612878900 end_time: 2021-02-09T14:10:00+00:00 = 1612879800

Da der Scheduler auch eine Zeit benötigt, um die Clip-Anfrage zu planen, können wir den gleichen Wert verwenden wie endzeit da es auch Unix-Zeit in Sekunden ist.

Wir wollen auch ein Videocloud-Video für den Clip erstellen, also binden wir das in die Ausgang. Ein Videocloud-Video wird zum Zeitpunkt der Planung des Workflows erstellt und seine ID wird in der Antwort zurückgegeben, damit Sie die Video-ID kennen, bevor der Clip erstellt wird. Wenn Sie die Clip-Anfrage später abbrechen oder zum geplanten Zeitpunkt fehlschlagen, wird das Videocloud-Video NICHT von Ihrem Konto entfernt.

Anfrage

Um den Workflow zu erstellen, gehen Sie wie folgt vor: POST Anfrage:

https://api.bcovlive.io/v1/scheduler/clip

Überschriften

Inhaltstyp: Anwendung/json X-API-KEY: Ihr API-Schlüssel

Anfragetext

{
	"job_id":  "your Job ID",
	"description": "My Scheduled Clip",
	"clip": {
		"output": {
			"label": "My Scheduled Clip",
			"start_time": 1612878900,
			"end_time": 1612879800,
			"videocloud": {
				"video": {
					"name": "My Scheduled Clip"
				}
			}
		},

}
Beispielantwort 
{ "account_id": "your Account ID", "description": "My Tuesday Clip", "type": "clip", "job_id": "your Job ID", "clip": { "state": "pending", "output": { "label": "My Scheduled Clip", "start_time": 1612878900, "end_time": 1612879800, "videocloud": { "video": { "id": "70702010152202", "name": "My Scheduled Clip" }} }, "time_utc": 1612879800 } "workflow_id": "d44cf29f4f184757ac0995beb8a0097e", }

Beachten Sie, dass clip.output.videocloud.video.id enthält die ID des neu erstellten Videocloud-Videos. Bei time_utc die Videoquelle wird mit dem aufgenommenen Clip aktualisiert.

Aktualisieren eines Clip-Scheduler-Workflows

Sie können beide aktualisieren Ausgang und time_utc Ihres Clips. Diese können nur aktualisiert werden, wenn sich der Job im Status „Ausstehend“ befindet: "zustand": "ausstehend".

Nach dem vorherigen Beispiel möchten wir, dass der Clip 5 Minuten kürzer ist als geplant. Beim Aktualisieren der Ausgang Feld müssen Sie das vollständige Objekt angeben, nicht nur die vorgenommenen Änderungen. Dies beinhaltet die Videocloud-Video-ID, die vom obigen Aufruf zum Erstellen zurückgegeben wurde.

Beachten Sie, dass wir in diesem Beispiel beide aktualisiert haben endzeit und time_utc um die 5-Minuten-Verkürzung widerzuspiegeln.

Anfrage

Gehen Sie wie folgt vor, um den Workflow zu aktualisieren: STELLEN Anfrage:

https://api.bcovlive.io/v1/scheduler/clip/{workflow_id}

Abbrechen eines Clip-Planer-Workflows

Sie können den Clip mit a cancel abbrechen LÖSCHEN Anfrage.

Beachten Sie beim Abbrechen eines Clip-Workflows, dass jedes zum Zeitpunkt der Planung erstellte Videocloud-Video NICHT vom Backend entfernt werden. Sie müssen verwaiste Videos manuell aus Ihrem Videocloud-Konto entfernen.

Anfrage

Gehen Sie wie folgt vor, um den Workflow zu aktualisieren: LÖSCHEN Anfrage:

https://api.bcovlive.io/v1/scheduler/clip/{workflow_id}

Überschriften

X-API-KEY: Ihr API-Schlüssel

Anfragetext

Es gibt keinen Anfragetext für diese Anfrage.

Benachrichtigungen

Beim Erstellen des Workflows können Sie optional Benachrichtigungen konfigurieren. Sie müssen eine URL für unseren Service angeben POST zu. Wir werden eine Benachrichtigung senden, wenn der Scheduler den Clip-Aufruf tätigt. Sie können auch so konfigurieren, dass Sie eine Benachrichtigung erhalten n Sekunden vor dem Clip. Siehe die Live-Planer-Benachrichtigungen für mehr Details.

Beispielanfragetext 
{ "job_id": "Ihre Job-ID", "Beschreibung": "Mein geplanter Clip", "notification_url": "http://example.com/live/clip/callbacks" , "clip": { "output": { "label": "My Scheduled Clip", "start_time": 1612878900, "end_time": 1612879800, "videocloud": { "video": { "name": "My Scheduled Clip" } } }, "time_utc": 1612879800, "Benachrichtigung": 600 } }
