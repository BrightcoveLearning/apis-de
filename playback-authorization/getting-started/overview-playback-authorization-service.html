Einführung

Der Playback Authorization Service (PAS) von Brightcove bietet ein zusätzliches Maß an Sicherheit bei der Bereitstellung von DRM-geschützten Inhalten mit Dynamic Delivery. Dies ist besonders nützlich für Kunden, die den Zugriff auf ihre Inhalte kontrollieren und das unbefugte Teilen von Inhalten verhindern möchten.

PAS ermöglicht es Kunden, Zugriffstoken mit einer Reihe von Konfigurationsvariablen basierend auf ihrer eigenen Geschäftslogik, wie z. B. Betrachterberechtigungen, zu erstellen, zu löschen und auf die schwarze Liste zu setzen. Dadurch wird verhindert, dass Lizenzinformationen an Dritte weitergegeben werden.

Diese Funktion ist für Kunden mit Abonnement von Video-On-Demand (SVOD) verfügbar, die eine Middleware-Lösung zwischen ihrem Benutzerverwaltungssystem (UMS) und dem Wiedergabedienst von Brightcove implementieren möchten.

Kontoeinrichtung

Stellen Sie sicher, dass Ihr Konto für Dynamic Delivery eingerichtet ist. Wenden Sie sich dann an Ihren Account Manager, um Ihr Konto für den Playback Authorization Service von Brightcove zu aktivieren.

Implementierung

Der Wiedergabeautorisierungsdienst von Brightcove ist ein regional bereitgestellter Dienst. Kunden müssen die Zugriffsberechtigungen für Zuschauer über ein externes User Management System (UMS) verwalten und über eine Middleware-Anwendung in PAS integrieren.

PAS bietet Kunden zwei Anwendungsfälle:

  • Sichern von DRM-Lizenzanfragen
    Unter Verwendung von Token mit DRM-geschützten Inhalten autorisiert PAS Anfragen basierend auf Benutzer- oder Sitzungsinformationen, um den Zugriff auf die DRM-Lizenzen zuzulassen oder zu verweigern.

  • Verschlüsselungsschlüssel sichern
    Unter Verwendung von Token mit HLSe-Inhalten autorisiert PAS Anfragen basierend auf Benutzer- oder Sitzungsinformationen, um den Zugriff auf die Verschlüsselungsschlüssel zuzulassen oder zu verweigern und die Sicherheit bei der Standardverschlüsselung zu verbessern.

PAS ist keine Standardlösung. Der Kunde muss eine Middleware-Lösung zwischen einem User Management System (UMS) und dem Playback Authorization Service von Brightcove implementieren. Diese Integration kann vom Brightcove Global Services (BGS)-Team oder vom Kunden selbst durchgeführt werden.

PAS unterstützt die Möglichkeit, einen signierten Satz von Inhaltsschutzoptionen zu übergeben und eine DRM-Lizenz oder einen Advanced Encryption Standard (AES)-Schlüssel zu gewähren, wenn diese Schutzoptionen gültig sind.

Publisher erstellen a JSON-Webtoken (JWT). Dieses Token wird mit der Wiedergabeanforderung übergeben. Wenn das Token ungültig oder abgelaufen ist, wird der Zugriff auf den Inhalt eingeschränkt.

Der Tokenablauf muss angegeben werden und darf einen Wert von mehr als 30 Tagen haben.
