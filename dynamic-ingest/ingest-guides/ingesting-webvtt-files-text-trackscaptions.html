Aufstellen

Das Setup für dynamische Ingest-Anfragen ist gleich, egal ob Sie ein Video, Bilder, eine WebVTT-Datei oder alles davon aufnehmen:

Anfrage-URL
        https://ingest.api.brightcove.com/v1/accounts/account_id/videos/video_id/ingest-requests
Authentifizierung
Die Authentifizierung erfordert ein Zugriffstoken, das als a . übergeben wird Bearer Token in an Authorization Header: 
        Authorization: Bearer {access_token}

Hinweis zu S3

Wenn Ihr WebVTT aus einem geschützten S3-Bucket abgerufen wird, müssen Sie eine Bucket-Richtlinie festlegen, damit Video Cloud auf die Dateien zugreifen kann. Sehen Verwenden der dynamischen Aufnahme mit S3 für Details.

WebVTT-Dateien aufnehmen

Sie können Ihrem Video auch WebVTT-Untertitel hinzufügen oder sie mithilfe von Dynamic Ingest für ein vorhandenes Video hochladen:

Textspuren während der Videoaufnahme hinzufügen

Anfrage-URL

        https://ingest.api.brightcove.com/v1/accounts/account_id/videos/video_id/ingest-requests

Methode

POST

Beispielanfragetext

        {
            "master": {
                "url":"http://learning-services-media.brightcove.com/videos/mp4/Water-Droplet.mp4"
            },
            "capture-images": false,
            "text_tracks": [
                {
        "url": "http://learning-services-media.brightcove.com/captions/for_video/Water-in-Motion.vtt",
        "srclang": "en",
        "kind": "captions",
        "label": "English",
        "default": true
                }
            ]
        }

Hinzufügen/Konvertieren von SRT oder DFXP

Zum Nur Videos mit dynamischer Lieferung , können Sie SRT- oder DFXP-Dateien auf die gleiche Weise aufnehmen. Das Aufnahmesystem von Video Cloud konvertiert sie automatisch in das WebVTT-Format und fügt sie dem Video hinzu.

Beachten Sie, dass:
  • Die in der DFXP-Datei eingestellte Sprache xml:lang muss mit dem Wert im Ingest-Aufruf übereinstimmen für srclang. Zum Beispiel, wenn die srclang Wert in der Aufnahmeanfrage ist en , dann muss die DFXP-Datei enthalten xml:lang="en".
  • Benutzerdefinierte Formatierungen in SRT-Dateien (z. B. Fading) werden bei der Konvertierung in WebVTT aus der Spur entfernt.
        {
            "text_tracks": [
              {
                "url": "http://media.brightcove.com/captions/Water.dfxp",
                "srclang": "fr",
                "kind": "captions",
                "label": "français",
                "default": true
              }
            ]
          }

Textspuren zu bestehenden Videos hinzufügen

Das Hinzufügen von Textspuren zu bestehenden Videos ist identisch, außer dass Sie keine der Videoinformationen oder das Profil im Anfragetext benötigen.

Anfrage-URL

        https://ingest.api.brightcove.com/v1/accounts/account_id/videos/video_id/ingest-requests

Methode

POST

Beispielanfragetext

        {
          "text_tracks": [
            {
              "url": "http://learning-services-media.brightcove.com/captions/for_video/Water-in-Motion.vtt",
              "srclang": "es",
              "kind": "captions",
              "label": "Español",
              "default": true
            }
          ]
        }

Anmerkungen:

  1. Untertiteldateinamen sollten nur Buchstaben (Groß- oder Kleinbuchstaben), Zahlen, Bindestriche (-) und Unterstriche (_) enthalten.
  2. Die label für die Untertitel wird vom Brightcove-Player für das Untertitel-Sprachmenü verwendet. Es empfiehlt sich, die entsprechende Sprache für das Etikett zu verwenden. Zum Beispiel: français zum French und so weiter.
  3. Der Text der Untertitel muss in UTF-8 kodiert sein, damit er für Double-Byte-Sprachen wie Japanisch und Koreanisch richtig angezeigt wird.
  4. Der Standardwert ist nicht pro Track, sondern für alle Tracks derselben Sprache auf true gesetzt (srclang).
  5. Für Bilder und Textspuren werden nur öffentliche HTTP- oder HTTPS-URLs oder signierte S3-URLs unterstützt.
  6. Bei Textspuren fügt Dynamic Ingest sie der VTT-Liste des Videos hinzu bzw. hängt sie an, auch wenn es sich um eine Neucodierungsanforderung handelt. Sie sollten die verwenden CMS-API um sie zu verwalten (aktualisieren/löschen).
  7. Dynamic Ingest akzeptiert bis zu 100 Textspuren gleichzeitig.
  8. Obwohl "mime_type": "text/vtt" wird im Allgemeinen für WebVTT unterstützt, Dynamic Ingest tut es nicht erlauben derzeit dieses Feld für die Aufnahme von Textspuren.
