Einführung

Beim Erstellen einer Dynamic Ingest-Anfrage mit dem Dynamische Aufnahme-API oder beim Hochladen von Videos mit dem Upload-Modul müssen Sie ein Profil angeben, das die Anzahl und den Typ der Wiedergabeversionen bestimmt, die durch den Video Cloud-Aufnahmeprozess erstellt werden. Dieses Thema bietet detaillierte Informationen zu jedem der Dynamische Lieferung und Kontextbewusste Codierung (CAE) Profile, einschließlich HEVC (h.265) Profile.

Anmerkungen:
  • Context Aware Encoding und HEVC-Codierung sind kostenpflichtige Add-ons für Video Cloud. Wenn Sie diese Funktionen zu Ihrem Konto hinzufügen möchten, wenden Sie sich an Ihren Account Manager.

Eine Übersicht über Dynamic Delivery finden Sie unter Überblick über die dynamische Bereitstellung.

Eine Übersicht über die kontextsensitive Codierung finden Sie unter Überblick über die kontextbezogene Kodierung.

Eine Übersicht über HEVC finden Sie unter HEVC-Video

Anmerkungen:

  • Dieses Dokument zeigt Profile für die reguläre Dynamic Delivery und Context Aware Encoding, einschließlich HEVC-Profilen.
  • Aufgrund der unterschiedlichen Funktionsweise von statischen Dynamic Delivery- und Context Aware Encoding-Profilen sehen Sie in den Profildetails unten unterschiedliche Informationen:
    • Bei statischen Dynamic Delivery-Profilen wird eine Liste der enthaltenen Darstellungen angezeigt
    • Für kontextsensitive Codierungsprofile wird eine Tabelle mit dynamischen Profileinstellungen angezeigt, die verwendet werden, um zu bestimmen, welche Wiedergabeversionen erzeugt werden sollen
  • Standardprofile können dupliziert werden, aber nicht bearbeitet - Wenn Sie eine geänderte Version wünschen, müssen Sie ein benutzerdefiniertes Profil erstellen.

Es ist auch möglich, eigene benutzerdefinierte Aufnahmeprofile zu erstellen. Informationen finden Sie im entsprechenden Thema:

  • Erstellen von Aufnahmeprofilen für die dynamische Bereitstellung
  • Erstellen von Aufnahmeprofilen für die kontextsensitive Codierung

Wiedergaben

Einzelheiten zu allen verfügbaren Standarddarstellungen, die Sie in Ihren Aufnahmeprofilen verwenden können, finden Sie in den folgenden Tabellen.

Anmerkungen:
  1. Dies sind alle Wiedergabeversionen, die für die Verwendung in Dynamic Delivery-Profilen verfügbar sind. Einige von ihnen werden derzeit möglicherweise nicht in den Standardprofilen verwendet.
  2. Die in der Tabelle angezeigte Fragmentdauer und Segmentlänge spiegeln nicht die tatsächlich gelieferten Segmentlängen wider. Dynamic Delivery kann abhängig von den Bedingungen für eine bestimmte Wiedergabesitzung Segmente unterschiedlicher Länge liefern. Standardmäßig werden 10-Sekunden-Segmente für HLS und 6-Sekunden-Segmente für DASH geliefert. Sieh dir das an Hinweis unten für eine ausführlichere Erklärung von Fragmenten und Segmenten.

Anmerkungen
  • [1-1] Wenn Sie in einem Aufnahmeprofil auf eine Wiedergabe verweisen, Stellen Sie sicher, dass Sie dem Namen voranstellen default/ - zum Beispiel: default/audio128
  • [1-2] Ein Segment entspricht einer herunterladbaren Datei. Mit HLS können Sie beispielsweise die Datei alle 6 Sekunden segmentieren. Jedes Segment enthält genügend Informationen, um es unabhängig von anderen Segmenten dekodieren und abspielen zu können. Ein HLS-Manifest listet jeden Segmentpfad und seine Dauer auf.

    Ein Fragment bezieht sich auf Fragmente einer MP4-Datei. Eine nicht fragmentierte MP4 enthält Metadaten am Anfang oder Ende der Datei und erfordert, dass die Daten (FTYP , MOOV Atome) vor den Medien (MDAT Atom). Um das Streaming zu erleichtern, kann eine MP4-Datei fragmentiert werden, sodass die für die Wiedergabe notwendigen Metadaten und die Medien innerhalb der Datei in Fragmente zerlegt werden. Dies ermöglicht es den Spielern, auf einfache Weise nach bestimmten Medienstücken zu suchen und reibungslose Übergänge zwischen verschiedenen Wiedergaben zu ermöglichen. Die Fragmentierung wurde ursprünglich für DASH erstellt, wird aber jetzt auch von HLS unterstützt.

    Sie können Medien auch fragmentieren und dashieren, sodass ein Segment (eine Datei) mehrere Fragmente enthalten kann. Es ist üblich, die Segment- und Fragmentdauer gleich zu machen. Beachten Sie, dass duration und length beziehen sich auf dasselbe, wenn Fragmente und Segmente diskutiert werden.
