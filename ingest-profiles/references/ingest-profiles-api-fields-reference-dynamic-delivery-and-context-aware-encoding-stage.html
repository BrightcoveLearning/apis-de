    Referenz zu Feldern aufnehmen

    Dies ist ein Verweis auf Felder für Aufgabestprofile für Dynamic Delivery (DD) und Context Aware Encoding (CAE).

    Siehe auch Ingest-Profile-API-Felder Reference (Legacy-Intest) , wenn sich Ihr Konto auf dem Legacy-Ingest-System befindet.

    Profil-Felder

    Wenn ein Benutzer ein Profil programmgesteuert mit der Absicht abruft, es als Grundlage für ein neues benutzerdefiniertes Profil zu verwenden, muss der Benutzer die ID, version, date_created, date_last_modified entfernen und brightcove_standard-Feldern, und der Name muss innerhalb des Kontos neu und eindeutig sein.
    Profiloptionen auf oberster Ebene
    Eigenschaft Art Beschreibung DD CAE Default
    id string Die Profil-ID (schreibgeschützt) kein
    version ganzzahlig Die Profilversion (schreibgeschützt) kein
    account_id ganzzahlig Die Video Cloud-Konto-ID kein
    name string Der Profilname - muss innerhalb des Kontos eindeutig sein kein
    description string Die Beschreibung des Profils kein
    date_created integer (Epochen-Timestamp (Millisekunden) Datum Profil wurde erstellt (schreibgeschützt) kein
    date_last_modified integer (Epochenzeitstempel in Millisekunden) Datum, an dem das Profil zuletzt geändert wurde (schreibgeschützt) kein
    brightcove_standard Boolesch Schreibgeschützt - false für alle benutzerdefinierten Profile falsch
    digital_master Objekt Definiert, ob und wie ein digitaler Master archiviert wird {}
    dynamic_origin Objekt Definiert Darstellungen und Bilder {}
    renditions Array Array von Formatvariantenobjekten; wird nicht für DD oder CAE verwendet, daher ist es ein leeres Array für diese Profile [] [[

    Digitale Master-Felder

    Das digital_master Objekt definiert, ob und wie ein digitaler Master gespeichert wird, den Sie später für das Retranscodieren verwenden können. Weitere Informationen finden Sie unter Archivierungsmaster .

    Digitale Master-Felder
    Eigenschaft Art Beschreibung DD CAE Default
    rendition string Die Formatvariante zum Archivieren - entweder passthrough oder none sind die einzigen Optionen für DD- und CAE-Profile Durchreiche
    distribute Boolesch Ob der digital_master an das CDN gedrängt werden soll, um für die Wiedergabe verfügbar zu sein; diese Option funktioniert derzeit nicht für DD- oder CAE-Profile und wird auf zurückgesetzt, false wenn Sie es auf true falsch

    Dynamische Ursprungsfelder

    Dynamische Ursprungsfelder
    Eigenschaft Art Beschreibung DD CAE Default
    renditions Array Array von zu erzeugenden Wiedergaben; für CAE sollten Sie nur alle Audio-Formatvarianten einschließen, die Sie möchten. Sehen Sie sich die verfügbaren Formatvarianten an, aus denen Sie [] [[
    images Array Array von Bildobjekten für das Poster und das Vorschaubild [] [[
    dynamic_profile_options Objekt Definiert Formatvariantenoptionen für CAE-Profile {}

    Bild-Felder

    Bild-Felder
    Eigenschaft Art Beschreibung DD CAE Default
    label string Der Bildtyp: entweder poster oder thumbnail kein
    height Nummer Höhe des Bildes in Pixeln kein
    width Nummer Breite des Bilds in Pixeln kein

    Felder für dynamische Profiloptionen

    Die Felder für dynamische Profiloptionen enthalten Richtlinien für die Erstellung von Formatvarianten, wenn Sie Context Aware Encoding verwenden.

    Dynamische Profiloptionen
    Eigenschaft Art Beschreibung DD CAE Default
    video_codecs Array Bei Mixed-Codec-Ausgaben ein Array der Codecs, für die Formatvarianten erstellt werden sollen. Derzeit lauten die zulässigen Artikelwerte h264 und hevc [2-1] kein
    video_codec_options Array Bei gemischten Codec-Ausgaben ein Objekt, das die Optionen für jeden Codec angibt. Jede Eigenschaft ist ein Objekt mit dem Namen h264 oder hevc , seine Eigenschaften sind die festgelegten Beschränkungen für Formatvarianten für diesen Codec. Die zulässigen Eigenschaften für diese Objekte sind:
    • min_renditions erforderlich
    • min_bitrate
    • max_bitrate
    • max_first_rendition_bitrate
    • min_resolution
    • max_resolution
    • max_first_rendition_resolution
    Hinweis: Diese Einstellungen werden auch auf globaler Ebene verwendet. Wenn Sie Einstellungen sowohl auf globaler als auch auf Codec-Ebene verwenden, müssen die Werte kompatibel sein. Beispielsweise müssen auf globaler Ebene festgelegte max_-Formatvarianten größer oder gleich der Summe der min_rendition-Werte sein, die auf der Pro-Codec-Ebene festgelegt sind
    		 [2-2] kein
    min_renditions Number Die Mindestanzahl von Formatvarianten, die erstellt werden sollen (erforderlich) kein
    max_renditions Number Die maximale Anzahl von Formatvarianten, die erstellt werden sollen (erforderlich) kein
    min_resolution Objekt Die Mindestauflösung für Ausgabeformate kein
    min_resolution.width Nummer Die Mindestbreite für Ausgabeformate in Pixeln kein
    min_resolution.height Nummer Die Mindesthöhe für Ausgabeformate in Pixeln kein
    max_resolution Objekt Die maximale Auflösung für Ausgabeformate ist standardmäßig Quellenauflösung
    max_resolution.width Nummer Die maximale Breite für Ausgabeformate in Pixeln Breite der Quelle
    max_resolution.height Nummer Die maximale Höhe für Ausgabeformate in Pixeln Höhe der Quelle
    max_bitrate Nummer Die maximale Bitrate für Ausgabeformate in KBit/s Quell-Bitrate
    max_first_rendition_bitrate Nummer Die maximale Bitrate für die niedrigste Bitrate-Wiedergabe in KBit/s Quell-Bitrate
    max_frame_rate Nummer Die maximale Framerate für Ausgabeformate in fps 1 (maximal 5)
    keyframe_rate Nummer Die Anzahl der Keyframes pro Sekunde für Ausgabeformate (ein Wert von 0,5 würde also alle zwei Sekunden zu einem Keyframe führen; ein Wert von 3 würde zu drei Keyframes pro Sekunde führen) 0.5
    select_baseline_profile_configuration Boolesch Mindestens eine im Profil verwendete Formatvariante ist ein Basisprofil falsch
    watermarks Array Array von Wasserzeichenobjekten , die ein oder mehrere Wasserzeichen angeben, die den Darstellungen hinzugefügt werden sollen falsch

    Hinweise

  • [ 2-1] Erforderlich wenn vorhanden video_codec_options ist. Weitere Informationen finden Sie unter HEVC Video .
  • [ 2-2] Erforderlich wenn vorhanden video_codecs ist. Weitere Informationen finden Sie unter HEVC Video .

    • Wasserzeichen-Felder

    Wasserzeichenbilder werden in jeden Videoframe in der unteren rechten Ecke eingebrannt.

    Beachten Sie, dass Wasserzeichen in CAE-Profilen über die unten gezeigten Felder angegeben werden können. Wenn Sie einem statischen Dynamic Delivery Ingest-Profil Wasserzeichen hinzufügen möchten, wenden Sie sich an den Brightcove Support.
    Eigenschaft Art Beschreibung Default
    url string URL für das Bild kein
    width Zeichenkette[3-2] Breite des Wasserzeichens in Pixel oder Prozentsatz der Rahmenbreite - z.B. 20%[3-1] kein
    height Zeichenkette[3-2] Höhe des Wasserzeichens in Pixel oder Prozentsatz der Rahmenhöhe - z.B. 10%[3-1] kein
    x Zeichenkette[3-2] Abstand vom linken Rand des Videobilds zur Mitte des Wasserzeichens als Pixel oder Prozentsatz der Bildbreite kein
    y Zeichenkette[3-2] Abstand vom oberen Rand des Videobildes zur Mitte des Wasserzeichens als Pixel oder als Prozentsatz der Bildbreite kein

    Hinweise

    • [ 3-1] Wenn Sie nur die Breite angeben, wird das Bildseitenverhältnis automatisch beibehalten
    • [3-2] Obwohl die width , height , x , und y Felder sind Zeichenfolgenwerte; Sie enthalten Zahlen (Pixel oder Prozent)
