Einführung

Die Informationen in diesem Thema beziehen sich hauptsächlich auf benutzerdefinierte Aufnahmeprofile. Standardprofile alle Archivmaster (siehe unter)

Wenn Sie beim Aufnehmen eines Videos einen Master archivieren, können Sie das Video später erneut transcodieren, ohne es erneut aufzunehmen. In diesem Thema werden Ihre Optionen zum Archivieren Ihrer Master erläutert. Beachten Sie, dass das Archivieren des Masters bedeutet, dass Sie die Master-Videodatei in Ihrem Konto speichern, damit sie für die zukünftige Verarbeitung verfügbar ist. Sehen Erstellen von Aufnahmeprofilen für die dynamische Bereitstellung für mehr Informationen.

Beachten Sie, dass der digitale Master für das Video nicht entfernt werden, und Sie können das Video weiterhin neu transcodieren.

Einen Master archivieren

Um einen Master für Ihr Video zu archivieren, müssen Sie die digital_master Objekt in Ihrem Aufnahmeprofil:

    {
        "account_id": 3423487123048,
        "digital_master": {
            "rendition": "passthrough",
            "distribute": false
        },
        "name": "nondrm-default",
        ...
    }

Quelle oder Wiedergabe

Bei Dynamic Delivery-Profilen (einschließlich Context Aware Encoding) können Sie die Videoquelldatei als Master archivieren (mit anderen Worten: "rendition": "passthrough" in dem digital_master Teil des Profils). Der digital_master wird nicht für diese Profile verteilt (als verfügbare Wiedergabe auf das CDN übertragen), auch wenn Sie "distribute": true.

Bei Legacy-Aufnahmeprofilen können Sie Ihre Videoquelldatei als Master oder eine hochwertige Wiedergabe als Master verwenden.

Um die aufgenommene Videoquelldatei als Master zu verwenden, fügen Sie den folgenden Code in Ihr Aufnahmeprofil ein:

    "digital_master": {
        "rendition": "passthrough",
        "distribute": false
    }

Um eine Wiedergabeversion für den Master anstelle Ihrer Quelldatei zu verwenden (nur ältere Ingest-Profile), stellen Sie die rendition Bereich der digital_master widersprechen dem Referenz ID für die Wiedergabe:

    "digital_master": {
            "rendition": "av0",
            "distribute": true
        },
        "name": "nondrm-av0",
        "packages": [],
        "renditions": [
        {
            "media_type": "video",
            "reference_id": "av0", ...

Wenn Sie einstellen rendition zu none , es wird kein Master archiviert , und du wirst nicht in der Lage sein, das Video neu zu transkodieren. Sie können jedoch ersetzen das Video, indem Sie es erneut aufnehmen.

Verteilung

Die Verteilung funktioniert nur für Ältere Ingest-Profile.

Sie können wählen, ob der Master verteilt wird oder nicht. Verteilt bedeutet, dass der Master auf das CDN übertragen wird und als abspielbare Wiedergabe verfügbar ist. Wenn der Master nicht verteilt wird, wird er von Video Cloud archiviert, aber nicht per Push an das CDN übertragen.

Anmerkungen:
  • Wenn Sie eine Ihrer Wiedergaben als Master ausgewählt und eingestellt haben distribute zu false , wird diese Wiedergabe nicht auf das CDN übertragen und ist nicht für die Wiedergabe verfügbar
  • Wenn die Wiedergabe auf eingestellt ist passthrough und distribute zu true , der Master wird nur verteilt, wenn er im h264-Format vorliegt
  • Das Originalvideo kann modifiziert werden, um es über das Internet abspielbar zu machen. Der archivierte Master ist jedoch mit der Originaldatei identisch, außer in sehr seltenen Fällen, in denen wir ihn ändern müssen, um ihn zu transkodieren
  • Das Verteilen des Masters macht nicht Arbeit für Videos, die mit aufgenommen wurden Dynamische Lieferung Profile. Wenn Sie sich die Aufnahmeprofile von Dynamic Delivery ansehen, sehen Sie das distribute ist eingestellt auf false. Wenn Sie das Profil duplizieren und versuchen, es in Studio zu ändern, wird der Wert auf . zurückgesetzt false automatisch.

Um den Master an das CDN zu verteilen, stellen Sie "distribute": true in dem digital_master Objekt:

    "digital_master": {
        "distribute": true,
        "rendition": "passthrough"
    }

Um zu verhindern, dass der Master verteilt wird, stellen Sie distribute zu falsch:

    "digital_master": {
        "distribute": false,
        "rendition": "passthrough"
    }

Standardverhalten

Standardmäßig (wenn das Profil kein a . enthält digital_master Feld), die Dynamische Aufnahme-API und Modul hochladen verhält sich genau so, als ob im Profil Folgendes enthalten wäre:

    "digital_master": {
        "distribute": false,
        "rendition": "none"
    }

Mit anderen Worten, es wird kein digitaler Master gespeichert.

Wenn Sie Folgendes in Ihrem Profil haben:

    "digital_master": {
        "distribute": false
    }

... ohne rendition Feld, um anzugeben, was archiviert werden soll, das Quellvideo wird archiviert als digitaler Meister.
